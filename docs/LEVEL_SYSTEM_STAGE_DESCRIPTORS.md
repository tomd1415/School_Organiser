# Computing progression — Stage descriptors (extracted from the school marking system)

> **Source:** the school's GradesMarks export (`GradesMarksAndPercentageScores_Subject_821531_27_06_2026`),
> sheets *ICT Stage 1 – ICT Stage 14*. Only the **stage/descriptor structure** from the file header was
> extracted — **no pupil names, UPNs, marks or scores** are reproduced here. This document is the raw
> input for the **Stages & strands** level system ([PHASE_16_PLAN.md](PHASE_16_PLAN.md), 16A): it gives
> the **year-ladder scheme**'s stages and their per-criterion descriptors.
>
> **Cleaned for import (2026-06-27):** source typos fixed ("goalsals"→"goals", "Recognize"→"Recognise",
> double spaces collapsed, "ie"→"i.e.", a bracketed gap-fill and trailing full stops removed). Intentional
> repeated descriptors (the same skill revisited at a higher stage) are **kept** as distinct per-stage criteria.

## How this maps to the Phase 16 model

- Each **ICT Stage 1–14** = one **stage** on the `year_ladder` progression scheme (ordinal = the stage number).
- Each bullet under a stage = one **criterion** (`stage_criteria.descriptor`) the pupil must evidence.
- The **strand** grouping below classifies every criterion into the four National-Curriculum Computing
  strands, so the same descriptors can be laid out as a **Stage × Strand** grid. Strands are advisory —
  adjust before seeding.
- Note: Stages 1–5 are single-descriptor early-years/EYFS-style statements (one criterion each); the
  per-strand structure becomes meaningful from Stage 6 onward.

## Part 3 — GCSE strand note

For the **GCSE group** the same evidence is re-expressed against OCR J277 as **grade equivalents** in
**two strands — Programming (Paper 2)** and **Theory (Paper 1)** (see [PHASE_16_PLAN.md](PHASE_16_PLAN.md)).
Roughly, the **CS-programming** criteria above feed *Programming (Paper 2)* and the **CS-theory + IT +
Data + Online-safety** criteria feed *Theory (Paper 1)* — but the GCSE scheme uses its own grade-based
stages and OCR spec points rather than these year-ladder descriptors, so it is seeded separately.
