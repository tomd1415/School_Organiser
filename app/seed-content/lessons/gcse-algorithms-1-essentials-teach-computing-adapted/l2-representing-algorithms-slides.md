# Representing algorithms

## Today we are learning
- describe the difference between an algorithm and a program
- name the flowchart symbols
- build a flowchart from a description

> 🧑‍🏫 Read the "I can…" aloud. Vocabulary on board: algorithm · computer program · programming language · flowchart · written description.

## Starter — building Lego
![Numbered Lego building instructions, steps 1 to 9]({{res:l2-lego-instructions.png}})

Why can anyone follow these, even in another country?

> 🧑‍🏫 Clear, decomposed, ordered steps + pictures (a visual language). Same outcome no matter who follows them — like a good algorithm.

## Algorithm vs computer program  (I do)
An algorithm is the ordered steps. A program is that algorithm written in code a computer can run.

> 🧑‍🏫 Class discussion — these words get mixed up. A human needs less detail than a computer.

## One algorithm, many ways
The same algorithm can be a written description, a flowchart, OR code (e.g. Python).

> 🧑‍🏫 Designing the logic first (often as a flowchart) speeds up writing the code later.

## The flowchart symbols
![A flowchart with a start, a process, a decision and an output]({{res:l2-flowchart-symbols.png}})

Start/Stop · Process · Decision (yes/no) · Input/Output.

> 🧑‍🏫 Point to each symbol on the board. A decision ALWAYS has two ways out. Learners label these on the worksheet.

## Worked example — a decision  (we do)
Follow a flowchart through both branches of a decision.

> 🧑‍🏫 Predict the next part. Make sure they can follow both the yes and the no branch before the worksheet.

## Your turn — the dice-roll flowchart  (you do)
