# Mini project

## Today we are learning
- identify features of good and bad web design
- link pages stored in the same folder
- insert an image stored locally

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on board: web design · local · remote · folder.

## Starter — match the tags
Match each tag to what it does on the starter worksheet.

> 🧑‍🏫 Quick recall of L1–L2 tags before we build. Go over the answers together.

## The parts of a web page  (we do)
![A web page with its parts labelled: menu, header, content, footer]({{res:l3-website-anatomy-teach-computing.png}})

A clear page has a heading, a menu, content and a footer in the same place on every page.

> 🧑‍🏫 Point out each part. Good design = clear, consistent, easy to read.

## What does BAD design look like?
![A cluttered, hard-to-read old web page]({{res:l3-bad-website-teach-computing.png}})

Flashing adverts, clashing colours and broken links make a site hard to use.

> 🧑‍🏫 Calm, low-arousal version of the discussion — no need to show flashing animation, the still is enough. Sort good vs bad on the worksheet.

## Plan before you build
![A hand sketching a website wireframe with labels]({{res:l3-wireframe-sketch-teach-computing.png}})

A quick sketch shows where the heading, links, images and text will go.

> 🧑‍🏫 The plan does not need to be neat — it decides content and layout.

## Local files — keep them together  (I do)
![A folder holding the pages, images and CSS for a website]({{res:l3-folder-structure-teach-computing.png}})

A **local** link or image uses just a file name: `page2.html` or `cat.png`. A **remote** one uses a full web address.

> 🧑‍🏫 Live demo: save an image into the folder, link it with just its file name, refresh. Stress: the file must be in the same folder.

## Your turn — add and link a page  (you do)
Make `page2.html`, add a local image, and link your pages with a list.

> 🧑‍🏫 Support: copy the menu from page 1. Core: build unaided. Challenge: add a "home" button on every page. Likely error: link points to a file that is not in the folder → "check the file name and folder match exactly".
