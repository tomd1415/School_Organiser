## Starter — how does a page reach your screen?
![Four web browsers]({{res:l1-web-browsers-teach-computing.png}})

A web page is a plain text file. Your browser asks a server for it and draws it on the screen.

> 🧑‍🏫 Think · pair · share. Draw out: every browser can open the same HTML file. No single right wording — it is about the thinking.

## What is HTML?  (I do)
HTML uses **tags** to mark up parts of a page. A tag sits inside angle brackets: `<h1>` opens, `</h1>` closes. The `/` marks the closing tag.

> 🧑‍🏫 `<h1>` = big heading, `<p>` = paragraph. Only the text BETWEEN the tags is affected. Every opening tag needs a closing partner.

## Tags are like the parts of a newspaper  (we do)
![A newspaper with a title, headlines and columns of text]({{res:l1-newspaper-teach-computing.png}})

A newspaper has a title, headings and paragraphs. A web page uses tags for the same jobs.

> 🧑‍🏫 Map it: masthead → `<h1>`, headline → `<h2>`, story text → `<p>`. Makes the tags concrete.

## Your turn — build a page  (you do)
Type `<!DOCTYPE html>`, then `<html>`, `<body>`, a heading, a paragraph and a list.

> 🧑‍🏫 Support: "type THIS line" card. Core: build unaided. Challenge: add an ordered list. Likely error: forgetting the closing `</tag>` → fix-words "every tag needs a partner that closes it with a /". Paste code + screenshot on the activity worksheet.

## Ordered and unordered lists
`<ul>` makes a bullet list; `<ol>` makes a numbered list. Each item goes in `<li>`.

> 🧑‍🏫 Quick live demo: switch `<ul>` to `<ol>` and refresh — bullets become numbers.
