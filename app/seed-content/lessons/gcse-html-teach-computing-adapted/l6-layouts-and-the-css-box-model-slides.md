# Layouts and the CSS box model

## Today we are learning
- plan a layout with a wireframe and a house style
- describe the parts of the CSS box model
- use margin and padding to space out elements

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on board: box model · padding · border · margin · float.

## Starter — same site, different screens
A website can be opened on a monitor, a tablet or a phone. The layout has to cope.

> 🧑‍🏫 Think · pair · share. Link back to L2 accessibility — a clear layout helps everyone.

## House style — colours and fonts
![A fan of colour swatches]({{res:l6-colour-swatches-house-style-teach-computing.png}})

A house style is the set of colours and fonts used on every page, so the site feels consistent.

> 🧑‍🏫 Tie to good design from L3 — consistency makes a site feel finished.

## The CSS box model  (we do)
![The CSS box model: content, padding, border, margin]({{res:l6-css-box-model-teach-computing.png}})

Every element is a box: **content** in the middle, then **padding**, then **border**, then **margin**.

> 🧑‍🏫 Padding is space INSIDE the border; margin is space OUTSIDE it, pushing other boxes away. Demo on ncce.io/cssboxmodel.

## Your turn — space it out  (you do)
Add padding and margin to a div, then use float to place two boxes side by side.

> 🧑‍🏫 Support: change one padding value and watch it move. Core: pad and border a div. Challenge: float two columns. Likely error: mixing up padding and margin → "padding pads INSIDE, margin pushes OUTSIDE".
