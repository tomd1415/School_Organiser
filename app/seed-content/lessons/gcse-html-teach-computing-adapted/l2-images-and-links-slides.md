# Images and links

## Today we are learning
- explain what accessibility means
- add an image with the `img` tag
- create a hyperlink with the `a href` tag

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on board: image · hyperlink · accessibility · source (src).

## Starter — what is accessibility?
Accessibility means a website works for everyone, including people who use a screen reader or cannot see colours clearly.

> 🧑‍🏫 Think · pair · share on the starter worksheet. Optional: play the TCC accessibility video (teacher-played, external link in the source) up to 4:00 — it has sound, so it is your choice given the low-arousal default.

## Adding an image  (I do)
![Two photo frames standing for images on a page]({{res:l2-image-frames-teach-computing.png}})

`<img src="dog.jpg" alt="My dog">` — `src` is where the image is, `alt` is text for screen readers.

> 🧑‍🏫 Live code it. Common errors: forgetting the quotes, or typing `scr` instead of `src`. `img` has no closing tag — it is self-closing.

## Image attributes — src, alt, width
`src` = the file, `alt` = the description, `width` = how wide in pixels.

> 🧑‍🏫 Stress the link between `alt` and accessibility — it is read aloud by screen readers.

## Making a hyperlink  (we do)
`<a href="www.bbc.co.uk">BBC</a>` shows the words "BBC" and jumps to the site when clicked.

> 🧑‍🏫 The `href` is the address; the text between the tags is what the user clicks. You can even put an `<img>` inside an `<a>` to make a clickable image.
