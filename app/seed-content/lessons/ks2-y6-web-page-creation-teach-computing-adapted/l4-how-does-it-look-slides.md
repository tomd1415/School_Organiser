# How does it look?

## Today we are learning
- add content to my own web page
- preview what my web page looks like
- evaluate my page on different devices and make edits

> 🧑‍🏫 Read the three "I can…" aloud. Words on the board: home page · preview · evaluate · device · Google Sites. Recap: last lesson we found copyright-free images; today we build the page.

## Starter — the steps to build
![A web page being built in Google Sites]({{res:l4-made-page.png}})

- open Google Sites → name the site → header → layout → content

> 🧑‍🏫 Show the set-up videos. Pupils order the steps on the starter worksheet. Support pupils choose what the home page is and which tool we use.

## Build your page
- use your paper plan from last lesson
- add your website name, logo, header, text and pictures

> 🧑‍🏫 Give a long, calm block to build (about 20 mins). Distribute the paper plans. TA: prompt with the layout boxes; do not build it for them. Early finishers: the exploratory task (header colour, image carousel, map).

## Preview on devices
![Previewing on a phone, tablet and computer]({{res:l4-devices.png}})

- people use phones, tablets and computers
- screens are different sizes
- Preview shows your page on each one

> 🧑‍🏫 Demonstrate Preview. Likely belief: "it looks the same everywhere." Fix-words: "different SCREEN SIZES — so it can look different." Thumbs up/down on the phone view together.

## Evaluate and edit
- does it look good on each device?
- if not, what edit would fix it?

> 🧑‍🏫 Pupils tick each device and suggest one edit (e.g. smaller header). Challenge: fix text missing off the phone view. Show your work: screenshot the page or a preview.

## I can…
Tick your three "I can…". Tell me why we preview on different devices.

> 🧑‍🏫 Listen for "so everyone has a good page, whatever device they use." Movement break before next lesson.
