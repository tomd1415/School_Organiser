## Starter — good names
A convention is an agreed way of writing code so others understand it.
Give variables clear names: `score`, not `c`.

> 🧑‍🏫 Show the times-tables program on the starter worksheet. Ask pairs to rename `c` and `d`. Stress: conventions help the READER and the WRITER.

## What is a paradigm?
A paradigm is a **style or pattern** of programming.
- Procedural — built from functions
- Object-oriented — built from classes and objects

> 🧑‍🏫 The word may be new. Plain definition: "a way of thinking about how you build a program."

## Procedural programming — functions
A function is a named block of code you can call again and again.
Repeated code is moved into ONE function.

## A new style — object-oriented programming
OOP builds programs from **classes** (blueprints) and **objects** (things made from the blueprint).

![Two short Python classes shown on screens]({{res:l1-oop-class-example-on-screens.png}})
