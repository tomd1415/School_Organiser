## Data travels in packets
![Packets are like parcels]({{res:l4-packets-like-parcels-teach-computing.png}})

A message is split into small packets, like parcels. Each packet can take its own route.

> 🧑‍🏫 Optional teacher-played hook (motion + sound — your choice): "A Packet's Tale" — youtube.com/watch?v=ewrBalT_eBM (3m30s).

## Inside a packet — header and payload
- Payload = a piece of the message.
- Header = where it came from, where it is going (IP addresses), and the sequence number.

> 🧑‍🏫 The sequence number lets the packets be put back in order at the other end.

## IP addresses
An IP address is four numbers from 0 to 255, separated by dots — e.g. 192.168.5.43. It is unique to a device.

> 🧑‍🏫 Show "what is my ip address" in a search engine. Each device on the internet has one.

## Protocols — TCP and IP
- TCP splits the message into packets and puts them back in order.
- IP routes the packets using IP addresses.

> 🧑‍🏫 Protocol just means "rules". TCP also re-sends any lost packet.
