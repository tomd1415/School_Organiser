# Bitmap file size calculation

## Today we are learning
- calculate the file size of a bitmap image
- describe how the number of pixels and colour depth change the file size

> 🧑‍🏫 Read the "I can…" aloud. Vocabulary: bitmap · colour depth · file size · bits · bytes. Hand out both worksheets.

## Starter — recap metadata
What does metadata mean? Name one piece of metadata stored with a photo.

> 🧑‍🏫 Answers: data about the data; e.g. date taken, dimensions, colour depth. Recap: divide bits by 8 to get bytes.

## The formula  (I do)
- File size in bits = width × height × colour depth.
- Divide by 8 to change bits into bytes.

> 🧑‍🏫 Plain, slow. Keep the formula on the board all lesson. No calculators needed — the numbers are small.

## Worked example  (we do)
8 × 8 pixels, colour depth 2:
- 8 × 8 × 2 = 128 bits
- 128 ÷ 8 = 16 bytes

> 🧑‍🏫 Do this together on the board. Then ask the class for 16 × 16 × 2 (= 64 bytes) as a we-do.

## Your turn  (you do)
- Use the activity sheet to find the file size of each image.
- Always show the working: pixels × pixels × depth → ÷ 8.

> 🧑‍🏫 Likely error: forgetting to divide by 8 (answer left in bits). Fix-words: "bits to bytes, divide by eight." TA: prompt, do not do it for them.

## What changes the file size?
- More pixels → bigger file.
- Higher colour depth → bigger file.
- The saved file is a bit bigger still, because of metadata.

> 🧑‍🏫 Each pupil says one thing that makes a bitmap file bigger.

Tick your "I can…". Tell me the file-size formula.

> 🧑‍🏫 Note who divided by 8 every time without a reminder.
