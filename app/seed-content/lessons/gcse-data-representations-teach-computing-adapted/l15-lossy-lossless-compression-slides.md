# Lossy and lossless compression

## Today we are learning
- explain what data compression is
- explain why data is compressed
- define lossy compression and lossless compression

> 🧑‍🏫 Read the "I can…" aloud. Vocabulary: compression · lossy · lossless · algorithm. Hand out both worksheets.

## Starter — why make files smaller?
A phone is full of photos. Why make the files smaller? What is making a file smaller called?

> 🧑‍🏫 Answers: more files fit / faster to send; this is called compression.

## What is compression?  (I do)
- Compression reduces the file size by reducing the number of bits.
- Advantages: faster transfer, and more files fit in the same storage.
- A compression algorithm shrinks the file; a decompression algorithm opens it back up.

> 🧑‍🏫 Plain line: "compression squeezes the file smaller." Keep it concrete with the phone-photos example.

## Lossy compression  (we do)
- Lossy compression permanently removes some data.
- The opened file is only an approximation — never exactly the original.
- Used for photos (JPEG) and music (MP3), where small changes are not noticed.

> 🧑‍🏫 Example: a 34.8 MB bitmap becomes a JPEG about 90% smaller, and the eye barely notices.

## Lossless compression  (we do)
- Lossless compression loses no data at all.
- The file can be restored exactly.
- Used for PNG images, ZIP files and text.

> 🧑‍🏫 Likely error: thinking lossless can't shrink much, or mixing the two up. Fix-words: "lossy LOSES data; lossless LOSES none."

## When can't you use lossy?  (you do)
- You cannot use lossy compression on text — changing letters changes the meaning.
- Pairs do the card sort: lossy vs lossless.

> 🧑‍🏫 Each pupil gives one example of a lossy file type and one lossless file type.
