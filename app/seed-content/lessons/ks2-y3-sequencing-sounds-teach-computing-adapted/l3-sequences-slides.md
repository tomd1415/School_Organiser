# Sequences

## Today we are learning
- explain that a program has a start
- name three event blocks
- join blocks into a sequence
- build a project from a design

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: sequence · event · start · run the code · design. Same routine as every lesson.

## Joined makes a sequence
![Three blocks joined into a sequence]({{res:l3-joined-blocks-make-a-sequence-teach-computing.png}})

- separate blocks: click each one
- joined blocks: click once, all run in order
- joined blocks = a sequence

> 🧑‍🏫 Starter (10 min). Compare separate vs joined. The point: joined blocks run one after another from one click. On the worksheet pupils choose which is a sequence and why.

## Events start it
![The three event blocks]({{res:l3-three-event-blocks-teach-computing.png}})

- when green flag clicked
- when this sprite clicked
- when key pressed

> 🧑‍🏫 I-do → we-do. Every script needs an event hat to start it. Pupils match each event to what triggers it. Likely error: forgetting the event so nothing runs. Fix-words: "every script needs an event hat to start it."

## Build from a design
- plan three sprites
- each one: an event + a movement
- build it, run it, check it
- if it does not work, debug

> 🧑‍🏫 You-do (about 23 min). Plan on the worksheet first, then build in Scratch. TA: prompt, do not do it for them. Screenshot the running project.

## Sequences are everywhere
- getting ready in the morning is a sequence
- the steps happen in order
- swap two steps and it goes wrong

> 🧑‍🏫 Quick real-world link. Order the morning routine on the board. Reinforces "order matters" before next lesson's sounds.

## I can…
Tick your four "I can…". Name one event block.

> 🧑‍🏫 Plenary. Check each pupil can name at least one event block and say what starts it.
