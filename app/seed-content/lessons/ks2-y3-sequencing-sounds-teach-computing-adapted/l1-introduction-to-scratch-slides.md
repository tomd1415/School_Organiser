# Introduction to Scratch

## Today we are learning
- find the three areas of the Scratch screen
- name a sprite and the stage
- place a block and run it
- add a new sprite

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: Scratch · block · command · sprite · stage. Same routine as every lesson.

## You already know this
![A Bee-Bot floor robot]({{res:l1-beebot-top.png}})

- you have used Bee-Bot and ScratchJr
- they follow commands
- Scratch is new, but it also follows commands

> 🧑‍🏫 Starter (10 min). Quick think-pair-share: what is the same about Bee-Bot and Scratch? Both follow commands and must be run. Bee-Bot is a real robot you pick up; Scratch is on the screen.

## The Scratch screen
![The three areas of the Scratch screen]({{res:l1-scratch-screen.png}})

- Blocks — on the LEFT, the commands you can use
- Code area — in the MIDDLE, where you build
- Stage — on the RIGHT, where the sprite moves

> 🧑‍🏫 I-do → we-do. Point to each area. Then pupils label the screen on the activity worksheet. Likely error: calling the cat "the stage". Fix-words: "the cat is the SPRITE; the stage is the white box it stands on."

## Place a block and run it
- drag a `move` block to the code area
- click the block to run it
- try `turn` and `point in direction`
- put each block back when you are done

> 🧑‍🏫 You-do (20 min). Pupils experiment with three motion blocks. TA: prompt, do not do it for them. Returning blocks to the palette stops them joining by accident.

## Add a sprite
- find the add-sprite button (bottom right)
- add one new sprite
- take a screenshot for your worksheet

> 🧑‍🏫 Then screenshot the project onto the activity worksheet. Keep it calm — one new sprite is enough.

## I can…
Tick your four "I can…". Point to the blocks, the code area and the stage.

> 🧑‍🏫 Plenary. Check each pupil can point to the three areas and name the sprite.
