# Programming sprites

## Today we are learning
- choose blocks to make a sprite move
- predict which block moved a sprite
- match a motion block to what it does
- program more than one sprite

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: sprite · motion · turn · go to · glide. Same routine as every lesson.

## Which block moved the cat?
![The motion blocks you can use]({{res:l2-motion-blocks-teach-computing.png}})

- the cat moved across the stage
- which block (or blocks) did it?
- tick all that could be right

> 🧑‍🏫 Starter (10 min). On the starter worksheet pupils tick ALL the blocks that could have moved the cat — more than one can be right (e.g. move, or go to).

## One block, one outcome
- move 10 steps — go forward
- turn 15 degrees — spin a little
- point in direction — face a way
- go to / glide — jump or slide to a spot

> 🧑‍🏫 I-do → we-do. Each block does ONE thing. Likely error: thinking turn moves the sprite to a new place. Fix-words: "turn changes the way it FACES, not where it is."

## Each sprite has its own blocks
- a block only controls its own sprite
- put blocks on Sprite 1 AND Sprite 2
- make each one move its own way

> 🧑‍🏫 You-do (20 min). Pupils match block ↔ action on the worksheet, then program two sprites. TA: prompt, do not do it for them. Then screenshot.

## Predict
- look at a sprite that has moved
- choose the block(s) that did it
- run it to check

> 🧑‍🏫 Quick prediction round. Run each to confirm. Praise good reasoning, not just the right answer.

## I can…
Tick your four "I can…". Name one block and what it does.

> 🧑‍🏫 Plenary. Note who can predict a block from a movement.
