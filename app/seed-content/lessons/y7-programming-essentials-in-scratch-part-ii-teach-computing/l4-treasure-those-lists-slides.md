# Treasure those lists!

## Today we are learning
- say what a list is
- describe why a program needs lists
- identify when a list is used in a program
- use a list (add, delete, replace, check)

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on board: list · variable. A list = a collection of related items kept under one name.

## Starter — which loop draws a pentagon?

## Why we need lists  (I do)
A variable holds **one** value — each new value replaces the old one. A list holds **many**.

> 🧑‍🏫 Demo the shopping-list project (ncce.io/shoppinglist). Trace the `item` variable on the board: eggs → cheese → flour. Each one overwrites the variable, but all three stay in the `shopping_list`.

## Lists in a game  (you do)
![The "Legends of Scratch" game with its inventory list]({{res:l4-legends-of-scratch-inventory-list-teach-computing.png}})

Play for five minutes, then investigate how the inventory list works.

> 🧑‍🏫 Be strict on the five minutes — the aim is to understand the code, not just play. Items get ADDED (apple), DELETED (watermelon to monkey), REPLACED (apple→diamond), and CHECKED (goblin, centaur).

## Add an item yourself  (you do)
Make the cake add itself to the inventory when it touches the cat — use the apple code as a guide.

> 🧑‍🏫 Likely error: confuses "replace item" with "add" → fix-words: "add puts a NEW item on the end; replace swaps the one already at that position." They paste a link + screenshot on the activity worksheet.
