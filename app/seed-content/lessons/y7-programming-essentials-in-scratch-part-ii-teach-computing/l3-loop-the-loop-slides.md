# Loop the loop!

## Today we are learning
- describe what each type of loop does
- say if a loop is count-controlled or condition-controlled
- evaluate which loop is best for a task
- justify my choice of loop

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on board: iteration · count-controlled · condition-controlled. Three loops: forever · repeat (n) · repeat until.

## Starter — describe these loops
![Scratch code: a count-controlled repeat 4 loop drawing a square]({{res:l3-count-controlled-repeat-loop-square-teach-computing.png}})

`forever` = until the game stops · `repeat (4)` = a set number of times · `repeat until` = until a condition is true.

> 🧑‍🏫 Forever and Repeat until are condition-controlled; Repeat (n) is count-controlled. Clarify before showing the objectives.

## Investigate — Tera's adventure  (we do)
Find the loops in a finished game and see what job each one does.

> 🧑‍🏫 Pair programming, swap roles every 5 minutes. They don't need to understand the whole game — targeted questions on individual loops.

## Evaluate — which loop is best?  (you do)
![Three Scratch loop options for drawing one square]({{res:l3-evaluate-the-loop-three-options-teach-computing.png}})

Read each scenario. Choose the best loop and say why.

> 🧑‍🏫 Scenario 1 (one square) → `repeat (4)`: it draws the shape once (forever wouldn't stop; repeat-until is over-complicated here). Do the first one together if needed.

## Justify your choice
For each scenario, write why you did or did not pick each loop.

> 🧑‍🏫 Likely error: picks `forever` for everything → fix-words: "forever never stops — only use it when the code should run the whole game." Support: choose from the picture. Challenge: invent a scenario for each loop.

## I can…
One fact about loops — tell me one thing you know about any loop.

> 🧑‍🏫 Plenary: each pupil states one fact; listen for gaps. Review the summary slide.
