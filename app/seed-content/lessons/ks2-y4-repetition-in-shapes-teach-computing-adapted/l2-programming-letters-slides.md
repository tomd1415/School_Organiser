# Programming letters

## Today we are learning
- plan an algorithm for a letter
- write the algorithm as Logo code
- test the code and debug it

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: algorithm · debug · bug · template. Same routine as every lesson.

## Debug my code
- this code should draw a **7**
- `FD 100  RT 9  FD 200`
- the turn is too small — what is wrong?

> 🧑‍🏫 Starter. Answer: RT 9 should be RT 90. "Debug" = find and fix the mistake. This matches the starter worksheet. Keep it calm and short.

## An algorithm is a plan
- an **algorithm** is a set of ordered steps
- we plan the steps first
- then we turn the steps into code

> 🧑‍🏫 Link back to Lesson 1. Stress: plan first, then type. A clear plan makes debugging easier.

## Squared-off letters
![Typing a command into the Logo command box]({{res:l2-fmslogo-typing.png}})

- use straight lines and 90 degree turns
- think: line, turn, line, turn
- start at the top and work down

> 🧑‍🏫 Show squared-off letters (like a digital display). Suggest letters that only need 90 degree turns (L, T, E, H). Some letters (K) need 45 degree turns — keep it simple.

## We do — the letter L
- `FD 200  RT 90  FD 100`
- line down, turn right, line along
- that draws an **L**

> 🧑‍🏫 Draw it on the board as you read each command. This is the Support order task on the activity worksheet. Then run it in Logo.

## Your turn — your initial
- pick the first letter of your name
- plan the ordered steps (your algorithm)
- type it in Logo and test it

> 🧑‍🏫 Support: draw the letter L. Core: a 3–4 line letter. Challenge: their own initial. TA: prompt, do not type it for them. Use `home` and `PU` to start fresh. Likely error: forgetting a turn — fix-words: "line, turn, line, turn."
