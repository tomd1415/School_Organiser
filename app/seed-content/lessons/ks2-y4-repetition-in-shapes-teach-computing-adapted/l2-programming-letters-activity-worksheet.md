## What we are learning
Today I will plan an algorithm for a letter, then write it as Logo code and test it.

## What is an algorithm?
An **algorithm** is a set of ordered steps. We plan the steps first, then turn them into code.

| Question | Choose |
|---|---|
| Why do we plan an algorithm before we code? | (  ) so we know the steps in order (  ) to waste time (  ) it is not useful |

## 🟢 Support
Put these steps in order to draw the letter **L** (start at the top, facing down).

```order
FD 200
RT 90
FD 100
```

## 🟡 Core
Match each Logo command to the move it makes when drawing a squared-off letter.

| The move | Match the command |
|---|---|
| Draw a line down | (  ) FD 200 (  ) RT 90 (  ) LT 90 |
| Turn a square corner to the right | (  ) FD 200 (  ) RT 90 (  ) LT 90 |
| Turn a square corner to the left | (  ) FD 200 (  ) RT 90 (  ) LT 90 |
