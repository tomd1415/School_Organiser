# Creating a program

## Today we are learning
- design a program that uses count-controlled loops
- use my design to write a program
- debug my program to improve it

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: design · program · count-controlled loop · procedure · debug · evaluate. Same routine as every lesson. This is the project lesson — pull all the skills together.

## Matching designs
![A pattern of squares repeated and turned, in three colours]({{res:l6-rotated-squares-pattern.png}})

- this pattern is one square, repeated and turned
- a small change in the loop makes a big pattern
- which code made it?

> 🧑‍🏫 Starter. A pattern like `repeat 9 [square rt 40]`. This matches the starter worksheet.

## The brief — wrapping paper
- use **more than one shape**
- use at least one **count-controlled loop**
- you can use the same shape in different sizes

> 🧑‍🏫 Show the brief clearly. Keep choices small for low load: offer square + triangle as a safe default. This matches the activity worksheet.

## Step 1 — design
- sketch your pattern first (your plan)
- label the shapes and how many times they repeat
- a plan makes coding easier

> 🧑‍🏫 Model a simple plan: a row of squares, then a row of triangles. Pupils plan on the activity worksheet before they touch Logo.

## Step 2 — program it
- turn your plan into Logo code
- use loops and procedures
- use `setpc` to change colour

> 🧑‍🏫 Demonstrate making a `bigsquare` procedure and changing colour with setpc. Remind them of `home`, `PU` and `PD` to move without drawing.

## Step 3 — debug and improve
- run your program
- does it match your plan?
- find and fix any mistakes (debug)

> 🧑‍🏫 Debugging strategies: trace the code, read it aloud, break it into chunks. Likely error: a wrong angle or missing bracket — fix-words: "check it against your plan." Save a screenshot.

## Evaluate
- does it use more than one shape? ✓
- does it use a count-controlled loop? ✓
- what would you improve next time?

> 🧑‍🏫 Plenary uses the evaluate section on the activity worksheet. Celebrate the finished patterns — print or display them. Note who can design, program and debug independently.
