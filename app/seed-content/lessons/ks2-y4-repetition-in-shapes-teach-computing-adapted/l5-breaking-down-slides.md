# Breaking things down

## Today we are learning
- break a real task into smaller steps (decompose)
- make and use a procedure in Logo
- explain that the computer can call a procedure many times

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: decompose · procedure · call · loop. Same routine as every lesson.

## Recap — a loop
- a hexagon is `repeat 6 [FD 100 RT 60]`
- one repeated chunk: `FD 100 RT 60`
- repeated 6 times

> 🧑‍🏫 Starter recap from Lesson 4. Show the hexagon, ask for the loop. Keep it short.

## Decompose a task
- **decompose** = break into smaller parts
- "get ready for PE" = lots of small steps
- big jobs are easier in small chunks

> 🧑‍🏫 Build a class list of the steps for getting ready for PE (or washing hands). This matches the starter worksheet order task.

## A procedure is a named chunk
![The Logo procedure editor: to square / repeat 4 [fd 100 rt 90] / end]({{res:l5-procedure-editor.png}})

- starts with `to` and a **name**
- the commands go in the middle
- finishes with `end`

> 🧑‍🏫 Demonstrate making the `square` procedure live. The name (square) is for US to remember — the computer just needs something to call. This matches the activity worksheet.

## We do — call the procedure
- once made, type `square` to run it
- `repeat 20 [square rt 18]` makes a pattern
- calling = running the whole chunk again

> 🧑‍🏫 Run it: 20 squares turned 18 degrees make a flower pattern. "Call" = run the named chunk. This is the Challenge on the activity worksheet.
