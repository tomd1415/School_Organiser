## The web browser and the web server
- A web browser is software that reads a web page's code (HTML) and shows it as text and pictures.
- A web server is a computer that stores websites and sends a page when a browser asks for it.

> 🧑‍🏫 They work together: browser asks, server sends. Optional: right-click a page and "View page source" to see the HTML.

## HTTP and HTTPS
- HTTP sends data as plain text — readable if intercepted.
- HTTPS encrypts the data — kept secret. The "S" means secure.

![An HTTPS web address with a padlock]({{res:l6-https-address-bar-teach-computing.png}})

> 🧑‍🏫 Use HTTPS for passwords and payments (look for the padlock). HTTP is fine for just reading a page.

## URLs
A URL is a web address. In http://www.facebook.com :
- http = the protocol
- www = the World Wide Web
- facebook.com = the domain name

> 🧑‍🏫 The top-level domain (the last part) hints at the type of site: .com, .gov, .edu, .co.uk.

## Domain names and DNS
IP addresses are hard to remember. The domain name system (DNS) maps an easy domain name (bbc.co.uk) to the device's IP address.

> 🧑‍🏫 Link back to lesson 4 — every device on the internet has an IP address.
