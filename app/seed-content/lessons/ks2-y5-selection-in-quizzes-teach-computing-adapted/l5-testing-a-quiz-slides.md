# Testing a quiz

## Today we are learning
- build the first question of my quiz in Scratch
- test my quiz with a right answer and a wrong answer
- share my quiz and give kind feedback

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: implement · program · test · run · debug. Same routine.

> 🧑‍🏫 We-do: which program has an else? (A.) Which asks a question? (Both.) Keep it short and clear.

## Build the first question
- say a welcome
- ask the first question
- check the answer with if… then… else…
- add the TRUE and FALSE outcomes

> 🧑‍🏫 I-do: model building just the first question. Stop and test before adding more. There is a demo video in the lesson folder to watch first.

## Test it
![A quiz running on the Scratch stage]({{res:l5-quiz-stage.png}})

- run it with the RIGHT answer
- run it again with a WRONG answer

> 🧑‍🏫 We-do: the output must match the plan both times. Test after each question, not at the end.

## Debug it
- used if… then… else…?
- spelled the answer correctly?
- TRUE outcome in "then", FALSE outcome in "else"?

> 🧑‍🏫 Likely error: outcome in the wrong part of the block. Fix-words: "true goes in then, false goes in else." TA: prompt, do not do it for them.

## Share it
Swap devices. Try a partner's quiz. Leave one kind comment.

> 🧑‍🏫 You-do. Model a kind comment first. Movement break as routine.
