# Asking questions

## Today we are learning
- ask a question in Scratch with ask and wait
- use the answer as the condition
- make the program go two ways (true and false)

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: question · answer · condition · true · false · outcome. Same routine.

## Starter — binary questions
- a binary question has only TWO answers: yes or no
- the answer becomes the condition

> 🧑‍🏫 Use the yes/no sort on the starter worksheet. Keep examples concrete and short.

## Ask and wait
- the ask block waits for the user to type an answer
- no forever loop is needed here — ask already waits

> 🧑‍🏫 I-do: model the ask () and wait block. The program pauses until the user answers.

## The answer is the condition
![ask Would you like me to grow? if answer = yes then change size else say OK]({{res:l3-ask-grow.png}})

- yes → the TRUE outcome
- anything else → the FALSE outcome

> 🧑‍🏫 We-do: point at the ask, the condition (answer = yes), and the two outcomes.

## It must match exactly
![ask What's your favourite number? if answer = 10]({{res:l3-ask-number.png}})

- "10" is TRUE
- "ten" is FALSE — it does not match

> 🧑‍🏫 Likely error: pupils expect "ten" to count. Fix-words: "the computer checks the exact letters and numbers."
