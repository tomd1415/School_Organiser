- find a way my quiz could be better
- add setup so the quiz starts the same every time
- improve my quiz in Scratch

## The matching problem
- the user must type the answer EXACTLY
- "correct", "Correct!" and "right" are not the same to the computer

> 🧑‍🏫 Scenario: three users knew the answers but only one got them all right — the others typed them differently. Fix idea: tell users how to answer.

## What is setup?
![Setup blocks: go to x 0 y 0, set size 100, point in direction 90, set colour effect to 0]({{res:l6-setup-blocks.png}})

- setup runs BEFORE the quiz starts
- it gives every user the same start

> 🧑‍🏫 I-do: like setting up a board game each time. If an outcome changes the colour, "set colour effect to 0" must be in setup.

## Improve your quiz
- add a setup
- tell users how to type answers
- test as you go

> 🧑‍🏫 You-do. Calm independent work. TA: prompt, do not do it for them. Movement break as routine.

## Evaluate
- How did you meet the task?
- What worked well?
- What did you improve?
- What could be better next time?

> 🧑‍🏫 Plenary think-pair-share. A scoring system would use variables — that comes in Year 6. End of unit: recap the journey condition → if… then… else… → ask → quiz.
