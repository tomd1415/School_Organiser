# Create a program

## Today we are learning
- describe iterative testing and final testing
- sort data into erroneous, boundary, and normal
- plan a program from a structure chart
- create and test a program

> 🧑‍🏫 Read the four "I can…" aloud. This is a DOUBLE lesson (and a project) — take your time. Vocabulary on the board: iterative testing · final testing · erroneous · boundary · normal. Learning partners; check in every 15 minutes.

## Starter — what is iterative testing?
- **Iterative testing** = testing **as you build**, bit by bit.
- You have done this all unit — e.g. `add(3, 4)` to check a function.

> 🧑‍🏫 I-do. Draw out that they already test as they go. Now we name it: iterative testing.

## Final testing — three kinds of data
- **Normal** data — sensible, expected values.
- **Boundary** data — the edges of what is allowed.
- **Erroneous** data — wrong values that should be rejected.

> 🧑‍🏫 We-do. Use the card-sort on the worksheet. Stress: a good tester TRIES to break the program with erroneous and boundary data, not just normal data.

## Test tables
- A **test table** records: the test, the input, the expected output, and pass / fail.
- Test normal, boundary AND erroneous data.

> 🧑‍🏫 We-do. Complete a row together for the guess-the-number program, then they fill the table on the worksheet.
