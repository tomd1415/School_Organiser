# Moving a sprite

## Today we are learning
- an event makes an action happen
- find the event and the action in code
- choose sensible keys to move a sprite

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: event · action · sprite · key press · move. Recap: an event is a thing that happens (like a key press). An action is what the sprite does. Same routine as always.

## How do games control movement?
- a game controller
- the keyboard
- the arrow keys

![Four arrow keys: left, up, down, right]({{res:l1-arrow-keys-teach-computing.png}})

> 🧑‍🏫 Starter, calm and short. Ask how they move a character in a game they know. Steer to "keys on the keyboard". This is the choose-the-keys question on the activity worksheet.

## Event and action
![A Scratch block stack: when r key pressed, point in direction 90, move 10 steps]({{res:l1-event-and-action-block-stack-teach-computing.png}})

- yellow block = the **event** (press the r key)
- blue blocks = the **action** (point and move)

> 🧑‍🏫 I-do. Point at the yellow block: "this is the event — it waits for the r key." Point at the blue blocks: "this is the action — it runs after the event." This matches the sort task on the starter worksheet. Likely error: thinking the sprite moves on its own → fix-words "nothing happens until the event."

## Try the keys
- press the key
- watch the sprite move
- the event makes the action happen

> 🧑‍🏫 We-do. Open the shared project together. Everyone presses a key and watches one sprite move. Keep it one step at a time.

## Choose your own keys
- pick keys that are easy to find
- the arrow keys make sense for up, down, left, right
- say why you chose them

> 🧑‍🏫 You-do: each pupil chooses keys and tests them. Support: choose the up key only. Core: choose all four arrow keys. Challenge: explain why arrow keys are clearer than random letters. TA: prompt, do not do it for them.

## Show me
Tick your three "I can…". Show me your sprite moving with the keys you chose.

> 🧑‍🏫 Plenary. Each pupil names one event and one action on their screen. Note who linked an event to an action.
