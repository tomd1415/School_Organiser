# Internet addresses

## Today we are learning
- computers use agreed rules (protocols) to send data
- devices on the internet have addresses (IP addresses)
- a DNS turns a website name into an IP address

> 🧑‍🏫 Read the three "I can…" aloud, slowly. Words on the board: protocol · address · IP address · DNS. Same routine as every lesson.

## Starter — how we use the internet
![Words about the internet]({{res:l1-word-cloud.png}})

- every time we use the internet, data is sent between computers

> 🧑‍🏫 Ask "how do you use the internet?" Pupils tick what sends data on the starter worksheet. A paper book is the odd one out (it does not use the internet).

## A protocol is an agreed rule
![Two people talking]({{res:l1-communicating.png}})

- we follow agreed rules: one side of the corridor, lining up to leave
- an agreed rule = a protocol
- computers follow protocols too, so they understand each other

> 🧑‍🏫 Likely error: "a protocol is a computer." Fix-words: "a protocol is an agreed RULE." Link it to your class talking and listening rules.

## Getting to the right place — addresses
![A postbox]({{res:l1-postbox.png}})

- a letter needs an address: name, street, town, postcode
- every device on the internet has an address: an IP address
- IP address example: 93.184.216.34

> 🧑‍🏫 Build the postal analogy. The IP address is the device's "postcode" so the reply knows where to come back to. Support pupils choose what an IP address is on the activity worksheet.

## Names are easier — DNS
![An address book]({{res:l1-address-book.jpg}})

- numbers are hard to remember; names are easy (bbc.co.uk)
- a DNS is like an address book: name → number
- it finds the IP address for the name you type

> 🧑‍🏫 Pupils order the DNS steps on the activity worksheet. Challenge: why is a name easier than a number? TA: prompt, do not do it for them.

## I can…
Tick your three "I can…". Tell me one thing: what does a DNS do?

> 🧑‍🏫 Listen for "name → number". Movement break before the next lesson.
