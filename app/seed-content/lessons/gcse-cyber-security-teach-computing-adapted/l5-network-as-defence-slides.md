# Network design as defence

## Today we are learning
- decide what a firewall should allow or block
- put the steps a firewall takes in order
- name ways to keep a network safe
- explain why a network needs more than one defence

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary: firewall · packet · network policy · MAC address filtering · user access levels · backup policy · antivirus.

## Starter — blocked websites
Some websites are blocked at school. Why might that be?

> 🧑‍🏫 Collect a few calm answers. Lead to: rules decide what the network allows.

## What is a firewall?  (I do)
A firewall is like a gate. It checks data going IN and OUT, and blocks anything that breaks the rules.

> 🧑‍🏫 Stress: it inspects traffic both ways. Show one allow and one block example.

## Allow or block?  (we do)
The rules (the network policy) decide. Students can move files under 5GB; gaming sites are blocked.

> 🧑‍🏫 Pairs sort the requests on the activity sheet, then explain their reasons. Then order the four firewall steps.

## More network defences  (you do)
- MAC address filtering — only known devices can join.
- User access levels — each person gets only what they need.
- Backups — spare copies so data can be restored after an attack.

> 🧑‍🏫 Pairs do the matching and the "tick all" task. Key idea: no single defence is enough on its own.

## Firewalls do not stop everything
A firewall does not stop a virus that a user lets in. That is why a network also needs antivirus and careful users.

> 🧑‍🏫 Link back to malware: a USB stick or email attachment can get past a firewall.
