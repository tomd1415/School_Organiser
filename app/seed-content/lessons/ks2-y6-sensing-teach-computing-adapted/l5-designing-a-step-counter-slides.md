# Designing a step counter

## Today we are learning
- choose the variable my step counter needs
- design the algorithm for my step counter
- design the program flow for my step counter

> 🧑‍🏫 Read the three "I can…" aloud. Words on the board: design · algorithm · variable · program flow · step counter. Recap: the accelerometer senses movement.

## Starter — counting steps
![The micro:bit, showing the accelerometer]({{res:l5-microbit-parts.png}})

- count your steps across the room
- a step counter does this for you
- it uses the accelerometer

> 🧑‍🏫 Pupils count steps across the room. Think-pair-share: which sensor could count steps? (the accelerometer). Support pupils choose what a step counter does.

## What a step counter does
![A step counter design: algorithm and program flow]({{res:l5-stepcounter-design.png}})

- a shake = a step
- button B shows the number of steps
- a message encourages you

> 🧑‍🏫 Read the task together. We are working at the DESIGN level today — planning, not coding yet. The code comes next lesson.

## Choosing the variable
- we need one variable: steps
- set steps to 0 at the start

> 🧑‍🏫 Core pupils name the variable (steps). Challenge: why start at 0? On the activity worksheet pupils fill in the variable section of their design.

## Designing the algorithm
- an algorithm is a plan written as clear steps, in order
- set steps to 0 → shake changes steps → button B shows steps → message

> 🧑‍🏫 Pupils order the algorithm steps on the activity worksheet, then write their own on the design template. TA: prompt, do not write it for them.

## Designing the program flow
- the program flow shows the path the program takes
- it follows the conditions you chose

> 🧑‍🏫 Pupils complete the program flow part of their design and trace it to check the order is right. Collect the finished designs — they are needed next lesson.

## I can…
Tick your three "I can…". Tell me the variable your step counter needs.

> 🧑‍🏫 Listen for "steps". Movement break before the next lesson.
