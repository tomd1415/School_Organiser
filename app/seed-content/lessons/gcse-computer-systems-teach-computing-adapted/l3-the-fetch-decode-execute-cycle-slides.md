# The fetch-decode-execute cycle

## Today we are learning
- name the three stages: fetch, decode, execute
- put the steps of the cycle in order
- describe what happens at each stage

> 🧑‍🏫 Read the "I can…" aloud. Vocabulary on the board: fetch · decode · execute · instruction · RAM · cycle. Same routine as every lesson.

## Starter — remember the CPU
- ALU = maths and logic
- control unit = in charge
- clock = sets the speed

> 🧑‍🏫 Quick retrieval of the CPU parts from last lesson. Matches the starter worksheet.

## Three stages
- FETCH — get the instruction from RAM
- DECODE — work out what it means
- EXECUTE — carry it out

> 🧑‍🏫 Say it as a rhythm: fetch, decode, execute… repeat. This is still the von Neumann model from last lesson. Likely error: "decode means run it" → fix-words "decode = WORK OUT, execute = DO."

## The cycle repeats
- after execute, the CPU goes back and fetches the NEXT instruction
- it does this millions of times a second

> 🧑‍🏫 This sets up the ORDER task on the worksheet. The clock speed sets how many cycles happen per second.
