# Main memory

## Today we are learning
- describe what RAM does and that it is volatile
- describe what ROM does and that it is non-volatile
- compare RAM and ROM
- define cache and describe its role

> 🧑‍🏫 Read the "I can…" aloud. Vocabulary on the board: RAM · ROM · cache · main memory · volatile · non-volatile. Same routine as every lesson.

## Starter — retrieval
- register = small fast store in the CPU
- ALU = maths and logic
- clock = cycles per second

> 🧑‍🏫 Quick retrieval of the CPU. Matches the starter worksheet multiple-choice questions.

## RAM
![RAM memory sticks]({{res:l4-ram-memory-sticks-teach-computing.png}})

- holds programs and data while they are RUNNING
- can be read AND written
- VOLATILE — loses everything when the power is off

> 🧑‍🏫 Teach "volatile" carefully and reuse the word often. Likely error: "RAM stores my files forever" → fix-words "RAM is emptied every time you switch off."

## ROM
- holds the START-UP instructions
- can only be READ
- NON-VOLATILE — keeps its data with the power off

> 🧑‍🏫 ROM = read only memory. It is why the computer knows how to start up. This sets up the RAM-vs-ROM sort task.

## Cache
![Memory chips on a circuit board]({{res:l4-memory-chips-on-a-circuit-board-teach-computing.png}})

- a small amount of VERY FAST memory
- sits between the CPU and RAM
- holds the data the CPU uses most, so it does not wait for RAM

> 🧑‍🏫 Use the "instruction cards" relay analogy if you have time: cache is close, RAM is far. Bring out HOW cache speeds the system up.
