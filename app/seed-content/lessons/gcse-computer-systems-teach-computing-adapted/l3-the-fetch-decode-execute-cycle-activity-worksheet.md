## What we are learning
Today I will put the fetch-decode-execute cycle in order and describe each stage.

## The three stages
The CPU runs every instruction in three stages: FETCH, then DECODE, then EXECUTE. Then it does the next one.

## Put the cycle in order
Drag the steps into the right order, starting with the first thing that happens.

```order
The instruction is fetched from RAM into the CPU
The control unit decodes the instruction (works out what it means)
The CPU executes the instruction (carries it out)
The result is stored if it is needed
The cycle repeats for the next instruction
```
