# Crunching numbers

## Today we are learning
- use arithmetic operators to calculate values
- read an assignment from right to left
- use `int()` to turn keyboard input into a number
- put program lines in the right order

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: operator · expression · assignment · integer · int(). Same routine as every lesson.

## Starter — predict
What is the value of `double`? What is `number`?

> 🧑‍🏫 Think · pair · share, then reveal. Misconception: an assignment is NOT an equation. `double` keeps the value it was given (10) even when `number` changes later.

## Maths in Python — the operators
`+` add · `-` subtract · `*` multiply · `//` whole-number divide

![Working out a value with an expression]({{res:l2-scratch-expression-teach-computing.png}})

> 🧑‍🏫 In Scratch you built sums from blocks; in Python you type the expression. Stress: Python works out the right-hand side first, then stores it.

## Order matters  (we do)
A variable must be given a value before it is used, or you get a NameError.

> 🧑‍🏫 Use the Parsons task on the activity worksheet. Lines run top to bottom — `secs` must be set before `mins = secs // 60` uses it.
