# How does a digital device work?

## Today we are learning
- a digital device takes an INPUT
- a digital device gives an OUTPUT
- input → process → output
- what makes a password strong

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: digital device · input · process · output · password. Same routine as every lesson. Calm, quiet room.

## What is a digital device?
![A desktop computer]({{res:l1-desktop-computer-teach-computing.png}})

- a digital device can do a JOB
- it does more than just turn on and off
- a computer, a tablet and a games console are digital devices

> 🧑‍🏫 Misconception: "anything electric is a digital device." A torch is electric but only turns on and off — it does not process. Keep it simple: "a digital device does a job."

## A digital device example
![A sat-nav (GPS)]({{res:l1-sat-nav-gps-a-digital-device-teach-computing.png}})

- you type where you want to go (input)
- it works out the way (process)
- it shows the map and tells you (output)

> 🧑‍🏫 Point to each part as you say input / process / output. Optional hand-jive: input = point down, process = roll hands, output = point out.

## Input → process → output
- **Input** = what goes IN (press a key)
- **Process** = the device does its job
- **Output** = what comes OUT (a letter on the screen)

> 🧑‍🏫 I-do: type a letter on the board computer. "Input = my finger on the key. Process = the computer works it out. Output = the letter appears." We-do: ask a pupil for another example.

> 🧑‍🏫 You-do: the `order` task. Likely error: swapping input and output — fix-words "input goes IN first, output comes OUT last." TA: prompt, do not drag it for them.

## Strong passwords
- a password keeps your device safe
- a strong password is LONG and HARD to guess
- use letters AND numbers

> 🧑‍🏫 A strong password is not your name or "password". Challenge pupils tick what makes a password strong. Keep it light — no real passwords shared aloud.
