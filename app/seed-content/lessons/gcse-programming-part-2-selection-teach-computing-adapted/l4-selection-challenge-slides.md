# The joke machine

## Today we are learning
- name the comparison operators
- choose the right comparison operator
- use selection in my own program
- build and test a joke machine in a pair

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: comparison operators · condition · selection · driver · navigator.

## Starter — comparison operators
![Keyboard keys: < and >]({{res:l4-comparison-operator-keys-teach-computing.jpg}})

`==`  `!=`  `<`  `>`  `<=`  `>=`

> 🧑‍🏫 Pupils match these on the worksheet. The first three are familiar; the last three are new. Predict from maths.

## Which one is "less than"?
The small, pointy end points at the **smaller** number: `3 < 7`.

> 🧑‍🏫 A memory trick: the arrow is like a mouth that eats the bigger number. Pupils write their own trick.

## Pair programming
![Two pupils at one computer]({{res:l4-pair-programming-teach-computing.png}})

- **Driver** types.
- **Navigator** reads the steps and checks for mistakes.
- **Swap every 5 minutes.**

> 🧑‍🏫 Set the pairs and roles before they start. Both must contribute. Use a 5-minute timer for swaps as a routine.

## The joke machine
It tells the start of a joke, takes a guess at the punchline, and gives a point if the guess is right.

> 🧑‍🏫 Demonstrate the finished game once so pupils know the goal.

## Build it step by step
Welcome → ask the guess → store it → check with `if` → score + feedback → reveal the score.

> 🧑‍🏫 The worksheet decomposes this into ordered steps with sample code and a common-errors checklist for each step.

## Keep score and give feedback
`if` correct → "Well done" and `score = score + 1`. `else` → feedback. `elif` for other wordings.

> 🧑‍🏫 Likely errors: one `=` instead of `==`; missing colon; print not indented; score not initialised at the top. Fix-words on the board.

## I can…
Tick your four "I can…". Show me your joke machine working.

> 🧑‍🏫 Note pairs whose game scores and gives feedback for right and wrong answers.
