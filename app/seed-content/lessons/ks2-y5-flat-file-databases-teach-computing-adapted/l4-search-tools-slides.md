# Search tools

## Today we are learning
- choose a field and a value to answer a question
- use AND to make a search narrower
- use OR to make a search wider

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: search · criteria · field · value · AND · OR. Same routine as every lesson.

## A record has fields
![One Titanic passenger record]({{res:l4-titanic-record.png}})

- the field names are on the left
- the values are in the middle
- some fields are text, some are numbers

> 🧑‍🏫 Point at fields (Age, Gender, Boarded) and their values. A search picks records by a field and a value.

## A simple search
- choose a field, e.g. Age
- choose a value, e.g. 16
- the database shows every match

> 🧑‍🏫 Unplugged "Simon Says": "stand up if you ate toast for breakfast." One rule = a simple search. Then live-search the Titanic database for Age = 16.

## AND makes it narrower
- AND means BOTH rules must be true
- fewer records match two rules
- e.g. Gender = Male AND Class = 1st

> 🧑‍🏫 Unplugged: "stand up if you have brown eyes AND ate toast." Fewer stand up. Misconception: "AND gives more" — fix-words "AND means BOTH, so FEWER match."

## OR makes it wider
![An OR search: Class = 1st OR Class = 2nd]({{res:l4-or-search.png}})

- OR means EITHER rule can be true
- more records match
- e.g. Class = 1st OR Class = 2nd

> 🧑‍🏫 Live-build the OR search. More records appear. Contrast with AND on the slide before.
