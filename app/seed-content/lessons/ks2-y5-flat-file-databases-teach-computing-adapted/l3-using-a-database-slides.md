# Using a database

## Today we are learning
- group records that share a value
- sort a group into order
- use group-then-sort to answer a question

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: group · search · sort · value · field. Same routine as every lesson.

## Why use a database?
- keeps a lot of data tidy
- lets us search fast
- lets us sort with one click

> 🧑‍🏫 Think-pair-share three reasons to use a computer database. Take answers, then show this list.

## Grouping the paper cards
- cut off the corners with no tick
- shake the cards
- you are left holding one GROUP

> 🧑‍🏫 We-do with your own animal cards from Lesson 1: hold the "lives in large groups" corner and shake. Link this to a computer search. TA: help them cut safely.

## Grouping on the computer (a search)
![The Countries database in table view]({{res:l3-countries-table.png}})

- search a field for a value, e.g. Language = German
- the database pulls out the matching group
- Germany, Austria and Switzerland are German-speaking

> 🧑‍🏫 Live-demo the search. Show the red cross clears it. Remind them of the more-than and less-than symbols for number fields.

## Group, THEN sort
![The Countries table sorted by population]({{res:l3-countries-sorted.png}})

- first GROUP (search Language = German)
- then SORT the group by Population
- the top row is the answer

> 🧑‍🏫 You-do on the activity worksheet. Likely error: sorting before searching → fix-words "group FIRST, then sort." TA: prompt, do not do it for them.
