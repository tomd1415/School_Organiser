# Computer databases

## Today we are learning
- name a field and a record in a database
- find data in form view and table view
- sort a database by a field

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: database · record · field · form view · table view · sort. Same routine as every lesson.

## The same words, on a computer
- a RECORD is one row (all the data about one thing)
- a FIELD is one column (one kind of data)
- the paper cards used the same words

> 🧑‍🏫 Bridge from Lesson 1: the cards become rows; the facts become columns. We use the Minibeasts database from J2e.

## The parts of a table
![A minibeast database in table view]({{res:l2-minibeast-table.png}})

- a FIELD is a column heading
- a RECORD is one whole row
- a VALUE is the data in one box

> 🧑‍🏫 Point at the heading row (fields) and one row (a record). This matches the label task on the activity worksheet. Misconception: "a column box is a record" — fix-words "a record is the WHOLE row."

## Form view and table view
![A record in form view]({{res:l2-countries-form.png}})

- FORM view = one record at a time, like a card
- TABLE view = every record in rows
- both show the same data

> 🧑‍🏫 Switch tabs live so they see the SAME data two ways. Form is good for reading one record; table is good for comparing many.

## Sort by clicking a field
- click a field heading once → sorts low to high (or A–Z)
- click it again → sorts high to low (or Z–A)
- use this to answer a question

> 🧑‍🏫 You-do: "Which minibeast has the most legs?" They sort the "How many legs" field. Likely error: clicking a row not the heading → fix-words "click the HEADING to sort." TA: prompt, do not do it for them.
