# Dream it up

## Today we are learning
- design a project for a real problem and audience
- break a project into separate features
- list the input, processing and output of a feature
- choose the hardware components we need

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary: design · problem · audience · decomposition · input · processing · output · prototype. Pairs work together from here to the end of the unit.

## Starter — input, processing, output
Every feature **takes input → does processing → gives output**.

> 🧑‍🏫 Use the starter sort task with a guessing game. Keep it concrete: a tilt is input, "make the range smaller" is processing, the number on screen is output.

## Four example projects
- Clicker — vote on a question (radio)
- Guess the number — the board guesses your number
- Music keys — cover a key to play a note (light sensor + radio)
- Password from gestures — unlock with a movement code

> 🧑‍🏫 These are starting points, not a menu — pupils may adapt or invent. Reassure: a SMALL, finished idea beats a big unfinished one.

## How to design  (we do)
1. problem  2. audience  3. features  4. input / processing / output  5. components

> 🧑‍🏫 Model this on ONE example out loud before pupils fill the proposal. Decomposition = break the big idea into small features you can build one at a time.

## You do — write your proposal
On the activity worksheet: pick an idea, order the design steps, then write your problem, audience and one decomposed feature.

> 🧑‍🏫 Support: matching project → component. Core: problem + audience. Challenge: decompose a feature. Take a photo of the proposal sheet for the screenshot box.

## I can…
Tick your four "I can…". Tell me your project in one sentence.

> 🧑‍🏫 A clear one-sentence pitch shows the design is focused. Keep proposals — they drive Lessons 5 and 6.
