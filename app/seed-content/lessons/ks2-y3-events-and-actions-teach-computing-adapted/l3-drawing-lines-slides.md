# Drawing lines

## Today we are learning
- use a programming extension (Pen)
- use pen down to draw a line
- choose blocks to set up my program

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: extension block · pen down · pen up · set up. Recap: last lesson we moved a sprite four ways. Same routine today.

## Extension blocks
- some blocks are hidden at the start
- click **Add Extension** (bottom left)
- choose **Pen** to get the Pen blocks

> 🧑‍🏫 I-do. Show where the Add Extension button is. This is the starter question on the starter worksheet. Keep it one step at a time.

## Pen down draws a line
- add **pen down** to the sprite
- now the sprite leaves a line as it moves
- "pen down" means the pen tip is on the screen

> 🧑‍🏫 We-do. Model pen down, then move the sprite — a line appears. Likely error: thinking "pen down" means put the pen on the table → fix-words "pen down means the pen draws."

## Set up the program
![Setup blocks: when green flag clicked, go to x 0 y 0, set pen colour to blue, erase all]({{res:l3-setup-blocks.png}})

- start in the same place every time
- set the pen colour
- clear the old lines with **erase all**

> 🧑‍🏫 This is the setup (initialisation). It runs under "when green flag clicked" so the program is the same every run. This matches the order task on the activity worksheet.

## You-do — add setup to your project
- add when green flag clicked
- go to x: 0 y: 0
- set the pen colour, then erase all

> 🧑‍🏫 You-do. Support: copy the setup blocks as shown. Core: choose your own pen colour. Challenge: explain why setup goes under the green flag. TA: prompt, do not do it for them.

## Show me
Tick your three "I can…". Show me a line your sprite drew.

> 🧑‍🏫 Plenary. Each pupil shows one drawn line. Note who added the setup blocks.
