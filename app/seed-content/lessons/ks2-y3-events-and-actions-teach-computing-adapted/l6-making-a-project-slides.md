# Making a project

## Today we are learning
- make design choices and say why
- build my design into a project
- test and evaluate my project

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: design · code · set up · test · debug · event · action. Recap: across this unit we moved a sprite, drew lines, added features, and debugged. Today we put it all together.

## Your own maze project
![A blank maze with a Start corner and a Finish corner]({{res:l6-blank-maze.png}})

- a sprite moves from Start to Finish
- it draws a line as it goes
- you choose the sprite, keys, and colour

> 🧑‍🏫 Introduce the task calmly. Show the maze. The project should reflect their own design.

## Design it first
- choose your sprite
- choose your keys (events)
- choose your pen colour

> 🧑‍🏫 I-do. Model one row of the design. This is the order task on the starter worksheet and the plan rows on the activity worksheet. A design makes coding easier.

## Build it in five steps
- add, resize and recentre your sprite
- choose your event blocks (keys)
- add the motion blocks
- add the pen blocks
- set up the project (x: -200 y: 100)

> 🧑‍🏫 We-do → you-do. Leave the five steps on screen. Likely error: forgetting the minus sign in x: -200 → fix-words "the start needs a minus sign for x." Support: follow a part-completed design. Core: full own design. Challenge: add an extra feature. TA: prompt, do not do it for them.

## Test and evaluate
- test every key
- debug anything that does the wrong thing
- say what works and what you would improve

> 🧑‍🏫 Test it against the design, like Lesson 5. Then pupils evaluate on the activity worksheet.

## Show me
Tick your three "I can…". Show me your finished maze project.

> 🧑‍🏫 Plenary. Each pupil shows their project and names one thing that works well.
