# Maze movement

## Today we are learning
- choose a character for my project
- choose a suitable size for the maze
- program movement in four directions

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: sprite · resize · move · point in direction · algorithm. Recap: last lesson an event made an action happen. Same routine today.

## The maze
![A maze with a sprite at the Start and a Finish corner]({{res:l2-maze.png}})

- start at **Start**, reach **Finish**
- do not touch the black lines

> 🧑‍🏫 Starter. Show the maze. Ask: would a big sprite fit these paths? This is the resize question on the starter worksheet.

## Choose and resize a sprite
- pick a sprite you like
- if it is too big, make the Size number smaller
- press Enter to set the size

> 🧑‍🏫 I-do → we-do. Model picking a sprite and changing the Size value. Likely error: a sprite too big to fit the paths → fix-words "make the Size number smaller, then press Enter."

## Program one direction
![Two block stacks: when up arrow point in direction 0 move 10; when right arrow point in direction 90 move 10]({{res:l2-four-direction-code.png}})

- event: when arrow key pressed
- action: point in direction, then move 10 steps

> 🧑‍🏫 We-do. Build the "right" stack together: when right arrow → point in direction 90 → move 10 steps. Test it. This is the order task on the activity worksheet.

## Copy it for all four directions
- right-click the stack → Duplicate
- change the key and the direction
- up = 0, right = 90, down = 180, left = -90

> 🧑‍🏫 You-do. Duplicate and change the copies. Support: do up only. Core: do all four. Challenge: change the steps value to move faster or slower. TA: prompt, do not do it for them.

## Did it work?
Tick your three "I can…". Show me your sprite moving four ways.

> 🧑‍🏫 Plenary. Ask each pupil to show one direction working. Note who resized and who programmed all four.
