## Mixing light — red, green, blue
Every colour on a screen is a mix of **red**, **green** and **blue** light. Each is set by an 8-bit number from 0 to 255.

> 🧑‍🏫 Optional demo: mix coloured light (additive). Mixing light makes things lighter; red + green light = yellow. Different from mixing paint.

## 24 bits for one pixel
- 8 bits for red
- 8 bits for green
- 8 bits for blue
- 8 + 8 + 8 = **24 bits per pixel**

> 🧑‍🏫 "RGB colour" almost always means 24-bit colour. Likely error: thinking RGB = 3 bits because there are 3 colours.

## How big is an image?
**resolution (pixels) × colour depth (bits) = size in bits**

![Rows × columns of pixels, each pixel's colour stored as bits]({{res:l2-resolution-colour-depth.png}})

> 🧑‍🏫 Worked example (I do): 800 × 600 = 480,000 pixels; × 24 = 11,520,000 bits. Then pairs do the worksheet table (we do → you do). Calculators allowed.

## Size vs quality
A bigger resolution looks sharper but needs more bits.

![A real photo — more pixels means a bigger file]({{res:l2-kingfisher.jpg}})

> 🧑‍🏫 Trade-off: more pixels / more colour depth = sharper but larger. A fast website wants smaller files.
