# Sonic playground

## Today we are learning
- calculate the size of a sound
- how sampling rate affects quality and size
- how sample size affects quality and size
- edit a sound using software

> 🧑‍🏫 Read the four "I can…" aloud. Starter recap on the worksheet: sample · sampling rate · sample size.

## Building the size formula
- count the samples each second (sampling rate)
- × bits per sample (sample size)
- × seconds (duration)
- × channels

> 🧑‍🏫 Build it one step at a time, describing the digitisation as you go. Calculators allowed.

## Worked example
1 second, 8000 samples per second, 1 byte per sample, 1 channel:
- 8000 × 1 byte × 1 = **8000 bytes** = 8 kilobytes

## Bigger or better — the trade-off
- more samples per second = better quality, bigger file
- more bits per sample = better quality, bigger file
