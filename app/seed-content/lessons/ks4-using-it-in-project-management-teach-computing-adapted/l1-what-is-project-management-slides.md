## What is project management?
![A project manager with a project board]({{res:l1-a-project-manager-with-a-project-board-teach-computing.png}})

- planning the work
- sharing out the tasks
- checking it gets done on time

> 🧑‍🏫 Project management decides if a project succeeds or fails. Today: four methods. Three came from software; one did not.

## Method 1 — Waterfall
![The waterfall method]({{res:l1-the-waterfall-method-teach-computing.png}})

- one step at a time: analyse → design → build → test → evaluate
- rigid; lots of planning at the start

> 🧑‍🏫 Likely error: thinking you can jump back a step. In pure waterfall you finish a step before the next. Fix-words: "one step at a time, in order."

## Method 2 — Agile
- flexible — work in small **sprints**
- you repeat steps and change things as you go

> 🧑‍🏫 Agile = agile/flexible. Good when the plan might change.

## Method 3 — Hybrid
- mixes waterfall planning with agile flexibility
- detailed start, flexible build

> 🧑‍🏫 Best of both. Plan carefully first, then build flexibly.

## Method 4 — Critical path
- came from building work (skyscrapers), NOT software
- shows which tasks MUST be done in order

> 🧑‍🏫 Plenary check: which method did NOT come from software development? Answer: critical path.
