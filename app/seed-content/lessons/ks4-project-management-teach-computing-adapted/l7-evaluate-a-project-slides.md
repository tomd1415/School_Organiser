# Evaluate a project

## Today we are learning
- recall the four stages of the project life cycle
- test a product using a testing table
- give and use peer feedback
- evaluate the success of the project

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: evaluation · testing table · feedback · recommendation. Same routine as every lesson.

## Starter — the project life cycle
- Initiate → Plan → Execute → **Evaluate**
- today we reach the last stage

> 🧑‍🏫 Pupils order the stages on the starter worksheet, then self-mark.

## Has the project been successful?
- look back at the feasibility report and SMART goals
- did you meet what the client asked for?

> 🧑‍🏫 Mark each product on the plan as done or not done.

## Testing tables
- a testing table records each test
- columns: the test · expected result · actual result · changes

> 🧑‍🏫 Likely error: only writing what happened, not what SHOULD happen. Fix-words: "expected first, then actual." Pupils write ten tests for the workbook.

## Peer feedback
- say what is good and why
- say what could be better and how

> 🧑‍🏫 Model kind, specific feedback. Not "it's rubbish" — say the fix.

## Final evaluation
- scope (what the project was) → each stage → recommendations
- **recommendations** = ideas to improve next time

> 🧑‍🏫 The final evaluation uses the testing tables and feedback. Screenshot the testing table onto the worksheet.

## I can…
Tick your four "I can…". Tell me one thing you would improve next time.
