# Improving and sharing

## Today we are learning
- find ways to make a game better
- add a 'lives' variable to a game
- share a game and give helpful feedback

> 🧑‍🏫 Read the four "I can…" aloud. Words on the board: improve · evaluate · share · lives. This is the last lesson — recap the whole unit at the end.

## Starter — are all games the same?
![Four different fruit catcher games]({{res:l6-four-different-fruit-catcher-games-teach-computing.png}})

- many people make catching games
- they all start the same but end up different
- you can borrow good ideas to improve yours

> 🧑‍🏫 Pupils tick ways to improve their game on the starter worksheet. "Delete the whole game" is the odd one out. Support choose one real improvement.

## Add a lives variable
![Scratch code that lowers a lives variable when a sprite is missed]({{res:l6-scratch-code-for-a-lives-variable-teach-computing.png}})

- **lives** counts how many tries are left
- **change lives by -1** when a sprite is missed
- you choose where the lives start (set lives to 3)

> 🧑‍🏫 Likely error: "change lives by 1 lowers it." Fix-words: "to go DOWN, change by -1." Pupils order the blocks, then add lives to their game.

## Share your game
- **save** your project
- click **Share**
- copy the web address (URL) and give it to a friend

> 🧑‍🏫 Pupils order the share steps. Remind them to copy the URL number carefully.

## Helpful feedback
- say what you **liked**
- say how it could be **even better**
- be kind and helpful

> 🧑‍🏫 Model a sentence starter: "It would be even better if…". Keep feedback positive and specific.

## End of unit — what we learned
- a variable is something that can change
- a variable has a name and a value, and holds one value at a time
- variables (score, lives, timer) make games better
- design → code → test → share

> 🧑‍🏫 Recap the journey: introducing variables → variables in programming → improving a game → designing → design to code → improving and sharing. Pupils self-rate their confidence (the scale) and tick the four "I can…".
