# Variables in programming

## Today we are learning
- a variable holds one value at a time
- a variable has a name and a value
- a good name says what the variable holds

> 🧑‍🏫 Read the four "I can…" aloud. Words on the board: variable · name · value · set · change. Recap: a variable is something that can change.

## Starter — a game with variables
![A Scratch game with a score and a timer]({{res:l2-score-timer.png}})

- the score changes; the timer changes
- both are variables

> 🧑‍🏫 Pupils tick the variables on the starter worksheet. The moving ball uses Motion blocks, not a variable — keep that aside for now. Support choose how many values a variable holds.

## One value at a time
- a variable holds **one value** at a time
- a new value replaces the old one
- the score can be 5, or 6, but not both

> 🧑‍🏫 Likely error: "it keeps both numbers." Fix-words: "the new value REPLACES the old one." Use a single drawer that can hold one thing.

## Name and value
- every variable has a **name** (Score)
- and a **value** (5)
- pupils sort names and values on the activity worksheet

> 🧑‍🏫 Name = what it is called; value = what it holds right now. Two parts, always.

## A good name
- a name should be **short**
- it should **say what it holds**
- **no spaces** — use _ (Home_team)

> 🧑‍🏫 Use the labelled drawers idea: a good label finds the pencils fast. Core: rename "Number" to something better.
