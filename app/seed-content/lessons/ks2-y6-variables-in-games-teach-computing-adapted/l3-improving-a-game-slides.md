# Improving a game

## Today we are learning
- choose variables to improve a game (score, timer, lives)
- set a variable to a start value with the green flag
- where in the code you change a variable matters

> 🧑‍🏫 Read the four "I can…" aloud. Words on the board: variable · set · change · design · event. Recap: a variable is a name and a value that can change.

## Starter — improving Pong
- the Pong game has a ball and a paddle, but no score
- you could add a **score**, a **timer**, or **lives**
- these are all variables

> 🧑‍🏫 Pupils tick variables to add on the starter worksheet. A "new background colour" is not a variable. Support choose which is a variable.

## Set the score at the start
![Pong code: when flag clicked, set score to 0]({{res:l3-pong-code.png}})

- **set score to 0** with the green flag
- this starts each game fresh
- **set** puts a value in; **change** adds to it

> 🧑‍🏫 Likely error: "change and set are the same." Fix-words: "set PUTS a value in; change ADDS to it." Pupils order the green-flag blocks.

## Where you change the score matters
- in the setup → the score changes once
- inside the loop → the score races up very fast
- inside "if touching paddle" → it goes up only when the ball is hit

> 🧑‍🏫 Pupils predict on the activity worksheet (the matching grid), then TEST each place in Scratch. Predict first, then run.

## Changing up and down
![A change score by 1 then change score by -1 block]({{res:l3-change-up-down.png}})

- a positive number makes the score go up
- a negative number (-1) makes it go down
- you can change a score in more than one place

> 🧑‍🏫 Core: if it goes up 1 then down 1, the score is the same. Challenge: why set to 0 at the start?
