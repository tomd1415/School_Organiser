# You've got the moves!

## Today we are learning
- say what a subroutine is
- say what decomposition means
- use subroutines to break a dance into moves
- make four subroutines in Scratch

> 🧑‍🏫 Read the four "I can…" aloud. Same routine as every lesson. Vocabulary on board: subroutine · decomposition · subproblem · call.

## Starter — what does this code do?
![Scratch code: two subroutines and a key press that calls one]({{res:l7-dance-battle-subroutines.png}})

Think · pair · share. A subroutine is a group of instructions with a name.

> 🧑‍🏫 A "custom block" in Scratch IS a subroutine. `define twist` makes it; pressing `a` CALLS it. Learners may not know direction blocks yet — show one quickly.

## We are subroutines (unplugged)  (we do)
Four volunteers each hold one move. When their name is called, they run it.

> 🧑‍🏫 Print the unplugged cards back-to-back: name faces the class, instructions face the volunteer. Call them in order, then at random. Point: once a subroutine works, you only need its NAME, not its inside.

## What is decomposition?
Breaking one big problem into smaller subproblems you can solve one at a time.

> 🧑‍🏫 The dance battle problem → one subroutine per move. Steer pairs to this if they don't reach it themselves. Likely error: "a subroutine runs on its own" → fix-words: "a subroutine only runs when it is CALLED."

## Make the subroutines  (you do)
Open the Dance Battle project. Make a custom block for each move, then call it with a key.

> 🧑‍🏫 Premade project (ncce.io/DanceBattle) so time goes on the subroutines, not art. They paste their link + a screenshot of the four subroutines on the activity worksheet.
