# Logging

## Today we are learning
- say what a data point is
- say what an interval is
- read a value from a data table
- read a value from a line graph

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: logging · data point · interval. Same routine as every lesson. Set a logger going at the start, so there is data to download later.

## Be the data logger
- watch the clip of a day
- write the temperature every hour
- each number is one DATA POINT

> 🧑‍🏫 Teacher plays the temperature clip (a beep marks each hour). Pupils record one number per hour. Pairs: one reads the screen, one writes.

## Interval
- the time between each reading is the INTERVAL
- here the interval is every hour
- a logger can use a much smaller interval

> 🧑‍🏫 Likely error: muddling "data point" and "interval". Fix-words: "a data point is a NUMBER; the interval is the GAP between them."

## Reading a table
![A data table]({{res:l3-data-table-of-readings-teach-computing.png}})

- each row is one data point
- the time goes up in small steps (0.1 s)
- read across to find the value

> 🧑‍🏫 We-do: find the sound value at 00.000 together (41.2 dBA). You-do: pupils read another value. TA: prompt, do not do it for them.

## Reading a line graph
![A line graph of light over time]({{res:l3-light-line-graph-over-time-teach-computing.png}})

- the data points join up to make a line
- low points = darker; high points = brighter
- the lowest dip is at about 3 seconds

> 🧑‍🏫 A line graph shows change over time at a glance. Support: point to the lowest and highest with a finger. Challenge: describe what happened between two times.

## I can…
Tick your four "I can…". Show me one data point on the graph.

> 🧑‍🏫 Note who can read a value from both the table and the graph.
