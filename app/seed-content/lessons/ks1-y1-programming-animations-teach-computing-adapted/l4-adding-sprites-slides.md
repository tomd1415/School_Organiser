# Adding sprites

## Today we are learning
- show that a project can have more than one sprite
- delete a sprite
- add blocks to each of my sprites

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: sprite · instructions · delete · program · algorithm. Recap: last lesson we changed a value.

## More than one sprite
![An underwater project with several sea-animal sprites]({{res:l4-an-underwater-project-with-several-sprites-teach-computing.jpg}})

- a project can have more than one sprite
- count them

> 🧑‍🏫 Hold up fingers for how many sprites. Each one is a separate character. This matches the count and sort tasks on the activity worksheet.

## Each sprite has its own program
- tap a sprite to choose it
- the programming area shows THAT sprite's blocks
- add blocks to each sprite, one at a time

> 🧑‍🏫 Key idea of the lesson: each sprite keeps its own instructions. Likely error: adds blocks but the wrong sprite is selected → fix-words "tap the sprite first, then add its blocks." TA: prompt, do not do it for them.

## Delete a sprite
- press and hold a sprite
- choose the X to delete it

> 🧑‍🏫 Model deleting. Reassure: if they delete the wrong one they can add it back. Calm, one step at a time.

## I can…
Tick your three "I can…". Show me a project with more than one sprite.

> 🧑‍🏫 Note who added blocks to each sprite and who could delete one.
