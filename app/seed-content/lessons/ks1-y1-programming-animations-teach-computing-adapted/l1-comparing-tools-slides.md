# Comparing tools

## Today we are learning
- find the commands to move a sprite
- use commands to move a sprite
- compare ScratchJr with a Bee-Bot

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: ScratchJr · sprite · command · programming area · compare. Same routine as every lesson. Last unit we used Bee-Bot floor robots; this unit we use ScratchJr on screen.

## Starter — this is ScratchJr
![The ScratchJr screen]({{res:l1-the-scratchjr-screen-teach-computing.jpg}})

- the cat is a SPRITE
- the blue blocks are COMMANDS
- the green flag runs the program

> 🧑‍🏫 Point at the cat (sprite), the blue blocks (commands), the green flag (run). On a tablet the app icon is the cat's face. This matches the label task on the starter worksheet.

## The move blocks
![Four move blocks]({{res:l1-four-move-blocks-teach-computing.png}})

- move right · move left · move up · move down

> 🧑‍🏫 Drag a blue block into the white programming area and TAP it to make the cat move. Misconception: the turn block does not move the cat, it spins it. Keep to the four move blocks today.

## Your turn — move the cat
- drag a move block down
- tap it
- can you make the cat go right? up?

> 🧑‍🏫 I-do first (model dragging one block and tapping). We-do one together. You-do: give time to try right, left, up, down. TA: prompt, do not do it for them. Likely error: drags the block but forgets to TAP it → fix-words "drag it, then tap it."

## Bee-Bot or ScratchJr?
- a Bee-Bot is a robot on the floor
- ScratchJr is on a tablet
- both follow commands you give

> 🧑‍🏫 Plenary discussion: same = both need commands in order; different = robot vs screen, floor vs tablet. This matches the Challenge on the activity worksheet.
