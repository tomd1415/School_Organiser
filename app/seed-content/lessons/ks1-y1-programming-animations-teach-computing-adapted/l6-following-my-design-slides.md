# Following my design

## Today we are learning
- use sprites that match my design
- add programming blocks based on my algorithm
- test the programs I have created

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: sprite · design · programming blocks · algorithm · test. Hand back each pupil's design and algorithm from last lesson.

## Recap — your design
![Three rockets on the moon, ready to race]({{res:l6-space-race.jpg}})

- you chose a background
- you designed three rockets
- you wrote an algorithm

> 🧑‍🏫 Quick recap of last lesson so everyone knows what they are building. A spare design handout is available for anyone who was absent.

## Step 1 — build your artwork
- choose your background
- design your three rockets to match your plan
- use undo/redo if you make a mistake

> 🧑‍🏫 Model block-filling a rocket to match the design. Reassure: mistakes are fine, undo/redo helps.

## Step 2 — add the blocks
- read your algorithm
- add the blocks to the matching rocket
- put them on the correct sprite

> 🧑‍🏫 Each rocket gets the blocks from its own algorithm. Likely error: blocks added to the wrong rocket → fix-words "tap the right rocket first, then add its blocks." TA: prompt, do not do it for them.

## Step 3 — test your programs
- run the program
- did the rocket do what you planned?
- if not, what will you change?

> 🧑‍🏫 Testing: run it, compare to the design, change if needed. This matches the testing questions on the activity worksheet. Support: did it work, yes/no. Challenge: why testing matters.

## Share your project
- show a partner or your teacher
- talk about what went well

> 🧑‍🏫 Calm sharing time. Celebrate finished projects. Recap the journey: design → algorithm → code → test.
