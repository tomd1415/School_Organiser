# Project design

## Today we are learning
- choose appropriate artwork for my project
- decide how each sprite will move
- create an algorithm for each sprite

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: sprite · background · appropriate · algorithm. Today is DESIGN only — no ScratchJr yet. We build it next lesson.

## Our project — a Space race
![Three rockets on the moon, ready to race]({{res:l5-three-rockets-a-space-race-teach-computing.jpg}})

- three rockets will race
- we plan it today

> 🧑‍🏫 Set the scene: a space race. Keep it calm and exciting, not over-stimulating.

## Step 1 — choose a background
- a planet, the sky, or the moon
- it must fit a space race

> 🧑‍🏫 Pupils sketch their chosen background. A quick sketch is fine — just enough to remember it next lesson.

## Step 2 — choose your rockets
- pick a colour for each rocket
- first, second, third

> 🧑‍🏫 Pupils colour and choose their three rockets. Solid colours so they can recreate them next lesson.

## Step 3 — how will they move?
- which way does each rocket go?
- up to the sky? across to a planet?

> 🧑‍🏫 Decide the path for each rocket. This becomes the algorithm.

## What is an algorithm?
- a precise set of instructions
- in the right order
- it can be turned into code

> 🧑‍🏫 Example algorithm: Start → Speed fast → Move up 6 → Stop. This matches the order task on the activity worksheet.

## Write your algorithm
- Start
- Speed fast
- Move up 6
- Stop

> 🧑‍🏫 You-do: pupils write an algorithm for each rocket. Support: order the given steps. Challenge: write their own. Keep all designs safe for next lesson. Likely error: an algorithm ScratchJr can't do → fix-words "only use moves you can find in ScratchJr."

## I can…
Tick your three "I can…". Show me your design and algorithm.

> 🧑‍🏫 Note who chose appropriate artwork and wrote an algorithm. Keep every sheet for Lesson 6.
