# Getting there

## Today we are learning
- explain what my program should do
- choose the order of commands
- debug my program when it goes wrong

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: plan · algorithm · program. Same routine as every lesson.

## Work it out
- last lesson we read and followed programs
- today we write and test our own
- first we plan, then we program

> 🧑‍🏫 Slide 3. Draw attention back to the prediction work. The new idea: plan first, then code.

## Getting from here to there
- turn over a start card and a destination card
- put the paper-bot on the start
- point out a route to the destination

> 🧑‍🏫 Activity 1. Pointing out the route is the first step of design — show intent before programming. No robot needed yet.

## Plan your algorithm
- choose command cards for your route
- put them in order along the route
- forwards/backwards cards go BETWEEN squares
- turn cards go INSIDE a square (turn on the spot)

> 🧑‍🏫 Activity 2. The ordered cards are the ALGORITHM — the plan. Command cards are a different colour from the robot buttons on purpose. Likely error: turn cards placed between squares — fix-words: "turns stay inside a square."

## Make it a program — and debug it
- press the buttons in the same order as your cards
- the algorithm is now a program
- if it goes wrong: clear, re-enter, watch each step, find the bug

> 🧑‍🏫 Activity 2 + plenary. Fixing a program is called debugging. Discourage picking the robot up — debug by watching each step. Start same square, same way, Clear first.

## I can…
Tick your three "I can…". Show me a program you planned and tested.

> 🧑‍🏫 Note who plans before pressing buttons and who can describe a bug they fixed.
