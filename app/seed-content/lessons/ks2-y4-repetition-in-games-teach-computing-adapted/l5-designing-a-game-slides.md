# Designing a game

## Today we are learning
- say where repetition is used in a game
- pick parts of a model game to use in my own
- write a simple design for my game

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: design · algorithm · model project · repetition · sprite · plan.

## The model game
![The model bat game]({{res:l5-bat-game.png}})

- the bats flap **forever**
- they keep moving down
- those parts **repeat**

> 🧑‍🏫 Starter. Play the model game. Pupils choose which parts repeat. Reassure: you do not build today — today is the plan.

## What is a good design?
- the **sprites** in the game
- what each sprite **does**
- which action **repeats** (the loop)

> 🧑‍🏫 I-do → we-do. A design = a plan before building. Pupils order the design steps on the design worksheet. Keep the plan small — one or two sprites is plenty.

## Write your algorithm
- when the green flag is clicked…
- my sprite will…
- the part that repeats is…

> 🧑‍🏫 You-do. Pupils fill the design worksheet: name sprites, describe the stage, write the algorithm for one sprite with a loop. Support: choose the repeating action. Core: fill the algorithm. Challenge: explain the game and why the loop is needed.
