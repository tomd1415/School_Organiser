# Creating our games

## Today we are learning
- follow my design to build my game
- find and fix a mistake in my code
- say how well my game works

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: build · algorithm · loop · fix (debug) · test · evaluate.

## Read your design first
![The bat game in the Scratch editor]({{res:l6-bat-game-in-the-scratch-editor-teach-computing.png}})

- open your project
- look at your plan from last lesson
- build one sprite at a time

> 🧑‍🏫 Starter. Pupils order the build steps. Calm, predictable. Remind: build a bit, then test — do not build everything before checking.

## Follow the algorithm
- add the sprite
- add its **loop**
- click the green flag and **test**

> 🧑‍🏫 I-do → we-do. Build one sprite together with its loop. Likely error: code runs once, no loop → fix-words "add the loop to repeat it."

## Test and fix
- run it — does it do what you planned?
- if not, find the mistake
- **fix** it and run again

> 🧑‍🏫 You-do. Support: choose what to do when the sprite does not move. Core: fill what you changed to fix it. Challenge: explain an improvement. TA: prompt, do not do it for them.

## Evaluate
- rate your game 1 to 5
- one thing that went well
- one thing to improve

> 🧑‍🏫 Pupils screenshot the finished game and self-rate on the evaluation worksheet. Keep it kind and low-stakes.

## Showcase
Show your game. Tick your three "I can…".

> 🧑‍🏫 Each pupil shows their game. Celebrate the loops they used. End of unit: real-life repetition → loops → shapes → animations → a game with repetition.
