# Modifying a game

## Today we are learning
- find the parts of a loop I can change
- say what my change does
- re-use code on a new sprite

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: game · sprite · forever loop · modify (change) · re-use.

## The shark game
![The shark game]({{res:l4-shark-game-on-the-stage-teach-computing.png}})

- each shark is a **sprite**
- a forever loop keeps it swimming
- we will **modify** it — change parts

> 🧑‍🏫 Starter. Pupils match each sprite to what it does. Modify = change something already there. We are not starting from scratch.

## What can change in a loop?
- the **number** (move 10 → move 5)
- the **block** inside the loop
- the **sprite** the loop is on

> 🧑‍🏫 I-do → we-do. Change "change x by 10" to "change x by 5" → slower. Likely error: deleting the forever block → fix-words "keep the loop, change what is inside." Pupils sort changes into faster / slower.

## Re-use the code
- add a **new sprite**
- copy the loop onto it
- now it moves too — no rebuilding

> 🧑‍🏫 Re-using saves time and avoids mistakes. Drag the script onto the new sprite in the sprite list. Show it working.

## Your turn
- change one number inside the loop
- run it and watch the difference
- screenshot before and after

> 🧑‍🏫 You-do. Support: choose what a smaller number does. Core: fill the effect of a change. Challenge: why re-using code saves time. TA: prompt, do not do it for them.
