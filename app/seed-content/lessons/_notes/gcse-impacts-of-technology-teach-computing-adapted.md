# Conversion notes — GCSE Impacts of technology (Teach Computing — adapted)

- **Slug:** `gcse-impacts-of-technology-teach-computing-adapted`
- **Course / key stage:** OCR J277 GCSE Computer Science · **KS4**
- **Source:** TeachComputing GCSE unit_11 (Impacts of technology, KS4 v1.2), Lessons 1–8 + the unit guide
  and the end-of-unit summative (questions + answers). © Raspberry Pi Foundation / STEM Learning, OGL v3.0.
- **Lessons authored:** 8 (each: starter worksheet + activity/quiz worksheet + slide deck; media where a
  suitable source image existed).
- **Self-verify:** PASS (`_x-verify.ts`, since deleted). Every declared file present; every `{{res:}}`
  placeholder maps to a declared file; slides titles end `.md`; every deck parses with ≥6 slides and
  teacher notes; every activity worksheet has a screenshot field; level slicing is distinct
  (support-only vs challenge-only content) on all starters and activities; all choice/multi-select cells
  have ≥2 options. The L8 quiz is deliberately a single shared assessment (no S/C/C — see below).
- **Question types used:** text, multiple-choice (single), multi-select, matching, fill-in-the-blank,
  card-sort, order/sequence, slider/rating, screenshot, checklist (10 interactive kinds — high variety).
- **Adaptation:** this is a discussion/theory unit (no programming tool), so "Show your work" was adapted
  to a written-answer text field + a 📷 photo-of-your-sheet cell. SEND defaults throughout: low-arousal
  decks, I-do/we-do/you-do, S/C/C on the same task, TA fix-words + likely-error on every activity, no
  flashing/sound, movement breaks as routine. Cohort-level only — no pupil is named or described.
- **Summative folded** into the L8 end-of-unit quiz: all 31 source MCQs reproduced as single-correct
  `choice` cells (one best answer each), grouped by impact area, plus the "explorer task" opinion question
  as a text field. Shared (no level sections) — everyone sits the same test; the L8 **starter** carries the
  lesson's S/C/C.

> **Heading gotcha avoided:** the words *support/core/challenge* appear ONLY in the literal level dividers
> (`## 🟢 Support` / `## 🟡 Core` / `## 🔴 Challenge`). Slide decks have a single `#` title and all `##`
> slides (no level dividers), so each deck is fully shared and parses cleanly.

a plain-git bundle (the databases unit flagged ~50 MB total for git-LFS; 206 MB on its own is impractical),

git-LFS lands (see `app/seed-content/lessons/README.md`), this clip could be added then. No other lesson had
an embedded source video (the L5/L7 AI, Simpsons and recycling clips are external YouTube links, kept as
teacher-played hooks in the notes).
