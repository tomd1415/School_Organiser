## A protocol is a set of rules  (I do)
A **protocol** is a set of rules for communication. The sender and the receiver both follow the same rules, so the message gets through.

> 🧑‍🏫 Plain line: "a protocol is the agreed rules so two devices understand each other." Agreed standards let devices from different makers work together.

## Sort the protocols  (we do)
Each protocol has a job: **web pages**, **email**, **files**, or **connecting the network**. Sort them on your worksheet.

> 🧑‍🏫 Do the first one together (HTTP → web pages). Then pairs sort the rest: HTTP/HTTPS=web, SMTP/POP/IMAP=email, FTP=files, Ethernet/WiFi=connecting. Ethernet = wired, WiFi = wireless.

## What the common protocols do
- **HTTP / HTTPS** — request and transfer web pages (HTTPS is the **secure** version)
- **SMTP** — *sends* email to a server · **POP / IMAP** — *fetch* your email
- **FTP** — uploads and downloads files
- **Ethernet** (wired) and **WiFi** (wireless) — connect devices to the network

> 🧑‍🏫 Mnemonic help: "S in SMTP = Send." HTTPS adds the **S for Secure** (it encrypts the page). POP downloads and removes; IMAP keeps mail on the server across devices.
