## What is a topology?  (I do)
A **topology** is the *shape* of a network — how the nodes are arranged.

> 🧑‍🏫 Plain line: topology = the shape. Sketch each shape on the board, one at a time, slowly.

## Star and bus
- **Star** — one central node (hub/switch); easy to add nodes; but the centre is a single point of failure.
- **Bus** — one cable with a **terminator** at each end; cheap and simple; but if the cable fails the whole network fails.

![Star and bus topologies, with their advantages and disadvantages]({{res:l4-star-and-bus-topologies.png}})

> 🧑‍🏫 Terminator = stops the signal bouncing back. Likely error: thinking the bus terminators are nodes — fix-words "terminators are caps, not computers."

## Ring and mesh  (we do)
- **Ring** — nodes in a circle; data goes one way around.

![A ring topology — nodes joined in a circle]({{res:l4-ring-topology.png}})

- **Mesh** — many paths; no central node, so no single point of failure; but lots of cabling = expensive.

![A mesh topology — many paths between nodes]({{res:l4-mesh-topology.png}})

> 🧑‍🏫 Sketch each shape on the board too. Ring = one loop, one direction. Mesh = lots of links, no centre. Likely error: confusing ring (a single loop) with mesh (many cross-links) — fix-words "ring is ONE circle; mesh is MANY links."
