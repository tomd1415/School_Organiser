## Client and server  (I do)
- A **client** asks for things (your phone or laptop).
- A **server** answers the requests and sends them back.

## Client–server vs peer-to-peer  (we do)
- Client–server: one strong central server; easy to back up and control; but the server is a **single point of failure** and costs more.
- Peer-to-peer: everyone equal; cheap and simple; but no central backup.

> 🧑‍🏫 Class sorts "client / server" jobs, then chooses the right model for the two scenarios (Core). Likely error: thinking the server is the same as a client — fix-words "server serves."
