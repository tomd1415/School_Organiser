# Network performance and routing costs

## Today we are learning
- describe the factors that affect network performance
- calculate how long a file takes to send
- find the lowest-cost route across a network

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary: bandwidth · latency · range · number of devices · transmission rate · routing cost. Hand out both worksheets.

## Starter — recap
Wired media = copper and fibre. Fibre = light, fast and far. Bluetooth = short range.

> 🧑‍🏫 Quick recall of transmission media. Pupils answer on the starter sheet.

## What affects performance?  (I do)
- **Bandwidth** — how much data fits through in a set time (like lanes on a motorway).
- **Latency** — the delay before data arrives.
- **Range** — how far a signal reaches before it weakens.
- **Number of devices** — more devices share the bandwidth, so each gets less.

> 🧑‍🏫 Use the concert photo: everyone sharing one connection = slow for all. The motorway-lanes idea is just an analogy — the cable doesn't get thicker.

## Calculating transfer time  (we do)
**time = file size (in bits) ÷ speed (in bits per second)**
Example: 50 bits ÷ 10 bits per second = **5 seconds**.

> 🧑‍🏫 Work the example slowly on the board, then pupils do the Core questions. Likely error: dividing speed by size — fix-words "bits on top, speed underneath." Calculators are fine.

## Routing costs  (you do)
▶ **Your turn — open your activity worksheet now.** Work out the transfer-time questions, then find the lowest-cost route.

Every link has a **cost**. The network picks the **lowest-cost route** — the path with the smallest total cost, not always the one with fewest links.

> 🧑‍🏫 Two routes on the board: 5+8+2 = 15 versus 4+5 = 9. Lowest = 9. Challenge pupils add up and choose. Optional: the Packet Tracer "investigating routes" videos — motion/sound, your choice.
