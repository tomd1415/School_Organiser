# Protecting a network

## Today we are learning
- say why network security matters
- name forms of attack on a network
- explain ways to keep a network safe

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary: phishing · pharming · social engineering · malicious software · anti-malware · DDoS · shouldering · blagging · brute force · user access rights. Keep the tone calm — about being careful, not scared. Hand out both worksheets.

## Starter — your data
School holds your name, address and grades. Why should that data be kept safe?

> 🧑‍🏫 Collect calm answers. Lead to: if a network is not protected, this data can be stolen.

## Two kinds of attack  (I do)
- **Social engineering** — tricking a person (phishing, blagging, shouldering).
- **Malicious software** — harmful code (viruses, spyware, a DDoS that floods a server).

![A padlock on a laptop keyboard]({{res:l12-padlock-laptop.png}})

> 🧑‍🏫 Plain line: one attacks the person, one attacks the machine. Class sorts the attacks on the worksheet.

## Pharming — a worked example  (we do)
Malware changes where a web address points, so a correct address sends you to a fake site that steals your details.

> 🧑‍🏫 Optional: the Packet Tracer pharming simulation. Likely error: thinking the user typed the wrong address. Fix-words: "the address was right; the signpost was changed." Class orders the four steps.

## Keeping a network safe  (you do)
▶ **Your turn — open your activity worksheet now.** Match each security method to what it does, then tick the ways to keep a network safe.

- Anti-malware software — finds and removes harmful code.
- Strong passwords — hard to guess or brute-force.
- User access rights — each person gets only the access they need.

> 🧑‍🏫 Pairs do the "tick all" task and the matching. Stress: people are often the weakest part — a logged-in PC left open is a risk.
