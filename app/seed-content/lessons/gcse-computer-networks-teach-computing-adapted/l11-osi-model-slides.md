# The OSI model

## Today we are learning
- name the seven OSI layers in order
- say what each layer does
- show how OSI relates to the TCP/IP model

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary: OSI model · application · presentation · session · transport · network · data link · physical · encapsulation. This is a short, calm lesson. Hand out both worksheets.

## Starter — recap TCP/IP
How many layers does the TCP/IP model have? Which layer adds the IP addresses?

> 🧑‍🏫 Answers: four layers; the internet layer adds IP addresses. Recap TCP = reliable.

## Seven layers instead of four  (I do)
The OSI model breaks the same job into seven layers, from the application at the top down to the physical at the bottom.

![The seven-layer OSI stack]({{res:l11-osi-layer-stack.png}})

> 🧑‍🏫 Read the seven names top to bottom slowly. Plain line: OSI is a more detailed map of the same journey. The class labels the stack on the worksheet.

## How OSI maps to TCP/IP  (we do)
- TCP/IP Application = OSI Application + Presentation + Session
- TCP/IP Transport = OSI Transport
- TCP/IP Internet = OSI Network
- TCP/IP Link = OSI Data link + Physical

> 🧑‍🏫 Class does the "group OSI into TCP/IP" sort. Stress: same journey, just split up more.

## Conceptual, not built that way  (you do)
The OSI model is **conceptual** — engineers use it to understand and find faults, layer by layer. Real networks are built on the TCP/IP model.

> 🧑‍🏫 Likely error: thinking OSI replaces TCP/IP. Fix-words: "OSI is the map; TCP/IP is the road we actually drive." Pairs do the matching and the "which are not application-layer" task.
