# Combining outputs

## Today we are learning
- connect more than one output component to a microcontroller
- use a count-controlled loop to control outputs
- design a sequence that uses a count-controlled loop

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary: output component · motor · repetition · count-controlled loop · algorithm. Same routine.

## Output components
- an **output** is a part the Crumble controls to DO something
- a **Sparkle** gives out light · a **motor** makes things spin

> 🧑‍🏫 Recap from last lesson. This matches the sort task on the starter worksheet.

## Connect a motor and a Sparkle
![Motor and Sparkle wired to the Crumble]({{res:l2-motor-sparkle-wiring.png}})

- the motor goes on the + and – pads near number 1
- the Sparkle stays on D

> 🧑‍🏫 Likely error: the motor spins the wrong way. Fix-words: "swap the red and black motor leads." The motor command can be forwards, backwards or stop, with a power %.

## Count-controlled loop
![A do 4 times program]({{res:l2-count-loop-code.png}})

- a **count-controlled loop** repeats the blocks inside it a set number of times
- "do 4 times" runs the sequence four times, then stops

> 🧑‍🏫 They met "repeat x times" in Scratch — same idea. The number is how many times it repeats.

## Algorithms guide our program
- an **algorithm** is a precise set of ordered steps
- we follow the algorithm to build the program

> 🧑‍🏫 Show a sketched algorithm and a written one. Discuss the steps before they build (this matches the order task).
