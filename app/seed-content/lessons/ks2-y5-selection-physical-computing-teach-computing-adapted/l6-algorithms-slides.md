# Writing and testing algorithms

## Today we are learning
- write an algorithm that describes what my model will do
- use selection to produce an intended outcome
- test and debug my project

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary: algorithm · selection · condition · action · test · debug. Final lesson — they need their model, design and circuit from last lesson.

## Clear and precise algorithms
- a vague algorithm: "make it spin"
- a precise algorithm: "spin the motor forwards at full speed for 2 seconds"

> 🧑‍🏫 Precise algorithms include timings, repeats, motor power and turning outputs off. This matches the order task on the starter.

## Write your algorithm
- use **selection**: if the switch is pressed, then start
- keep checking the condition (a forever loop)
- include waits, and turn all outputs off at the end

> 🧑‍🏫 Ask them to talk their algorithm through with a partner before coding.

## Build from the algorithm

- the code follows the algorithm, step by step
- test a few blocks at a time, then add more

> 🧑‍🏫 Good practice: run after a few blocks. This makes debugging easier — they find the bug sooner.

## Test and debug
- run the program and watch the model
- if it does not work, fix the code or the algorithm

> 🧑‍🏫 Likely error: the program never starts. Fix-words: "check the input — is the switch on the right pads, is the condition 'A is HI'?" Support: choose the bug.

## Evaluate
- did your carousel meet the task? (selection + two outputs)
- how would you improve it with more time?

> 🧑‍🏫 Think-pair-share. Guide improvements towards the algorithm or extra outputs, not just looks.
