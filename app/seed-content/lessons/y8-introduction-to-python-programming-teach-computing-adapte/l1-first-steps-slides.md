# First steps in Python

## Today we are learning
- explain the difference between an algorithm and a program
- write a Python program that prints a message
- use a variable and input to greet the user
- find and fix a syntax error

> 🧑‍🏫 Read the four "I can…" aloud. Same routine as every lesson. Vocabulary on the board: algorithm · program · variable · input · output · syntax error.

## How we work — in pairs
One person types (the driver), one person guides (the navigator). Swap after each task.

![Two pupils working together at a laptop]({{res:l1-pair-programming-teach-computing.png}})

> 🧑‍🏫 Pair programming. Set the pairs in advance. Remind them to swap roles each task — this keeps both pupils involved and lowers pressure.

## Starter — algorithm or program?
An algorithm is steps for a person. A program is steps for a computer.

> 🧑‍🏫 Use the card-sort on the starter worksheet. Keep it concrete: a recipe (algorithm) vs `print("Hello")` (program). A program must follow exact rules so a machine can run it.

## Our first program — Hello world
![The Hello world program in Python]({{res:l1-hello-world-task-teach-computing.png}})

> 🧑‍🏫 Live-code this on the board. Type it slowly. Point out the brackets and the quotation marks. Then make one deliberate mistake (capital P) and read the error message together.

## From Scratch to Python — input
In Scratch you used "ask and answer". In Python we use `input()`.

![Scratch ask and answer blocks]({{res:l1-scratch-ask-and-answer-blocks-teach-computing.png}})

> 🧑‍🏫 Link the familiar Scratch blocks to the new Python lines: `print("What's your name?")` then `name = input()`. The variable stores what the user types.

## Your turn — greet the user  (you do)
Write a program that asks for a name and says hello.

> 🧑‍🏫 Support: give a printed "fill the gaps" line. Core: write it unaided. Challenge: greet with two lines. Likely error: missing a bracket or quotation mark — fix-words "read the error, find the line, count the brackets."

## Fixing mistakes — syntax errors
A syntax error means one letter or symbol is wrong. The computer cannot guess what you meant.

> 🧑‍🏫 Reassure: everyone gets syntax errors — they are normal, not failure. Model reading the error message calmly and fixing one thing at a time.
