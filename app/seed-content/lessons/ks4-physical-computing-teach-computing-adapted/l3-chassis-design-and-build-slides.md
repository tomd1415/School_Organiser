# Chassis design and build

## Today we are learning
- plan where each component will sit
- choose materials strong enough for the job
- build a chassis that holds together when it moves
- test the buggy with a simple program

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary: chassis · prototype · structural integrity · aesthetics · power bank. Hands-on making lesson — minimal code today.

## Starter — what makes a good chassis?
- it must HOLD all the parts (motors, board, battery, sensors)
- it must STAY TOGETHER when the buggy moves

> 🧑‍🏫 Pupils sort strong vs weak chassis on the starter worksheet. Remind them of the considerations from the end of last lesson.

## Plan first
![Where the parts sit on a buggy]({{res:l3-buggy-component-layout-teach-computing.png}})

- sketch where each part will go BEFORE you fix anything
- keep the heavy battery low and central for balance
- one end needs a smooth third support, level with the wheels

> 🧑‍🏫 The third support need not be a wheel — any smooth surface that keeps that end the same height as the driven wheels.

## Build it
- mark out and cut the base
- attach the motors and wheels
- add the third support
- mount the Pico, breadboard and battery

> 🧑‍🏫 Pupils order these steps on the worksheet, then build. Support: a parts checklist. Core: build to plan. Challenge: tidy and decorate. TA: prompt for a smaller, sturdier base first.

## Test and reflect
- run your forwards/backwards code
- does it hold together and drive straight?
- untethered? you need a separate power bank

> 🧑‍🏫 Plenary: did it work as planned, what would you change? Note which buggies are stable enough to carry on. Screenshot/photo the chassis.
