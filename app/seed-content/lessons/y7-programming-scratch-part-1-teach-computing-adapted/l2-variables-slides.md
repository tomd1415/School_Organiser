# Variables in Scratch

## Today we are learning
- what a variable is
- input → process → output
- predict a program that uses a variable
- trace the value of a variable

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on board: variable · store · input · process · output · trace.

## Starter — variables storytime
A **variable** is a name that stands for some stored data.

> 🧑‍🏫 Read the story; pupils shout the word linked to each variable name. Change TOWN half-way through. Key point (misconception M1): a variable holds ONE value at a time — the new word replaced the old one.

## Input → process → output
A computer takes an **input**, **processes** it, and gives an **output**. It **stores** data to use later.

> 🧑‍🏫 Example: keyboard = input, screen = output. The variable is where the data is stored in between.

## Predict — Chat with Big Ed  (I do → we do)
![Chat with Big Ed: ask name, set name to answer, say join Hello name]({{res:l2-chat-with-big-ed-variables-blocks-teach-computing.png}})

The program asks your name, **stores** it in a variable, then says "Hello [name]".

> 🧑‍🏫 Predict, then run (PRIMM). The orange blocks are variable blocks. Likely error: thinks "Hello" shows without the name → "the variable `name` only fills after you type and it is set."

## Your turn — investigate and modify  (you do)
Open the program, investigate the variable blocks, then add the music question.

> 🧑‍🏫 All pupils do the investigate steps; modify is the stretch. Pair programming, swap roles every 5 minutes. Drag blocks, no typing of code.

## Trace it — What's the temperature?
Big Ed says the value of `temperature` each time it changes. Write each value down.

> 🧑‍🏫 Tracing shows the value changes during the run. Support: values are small whole numbers. Challenge: explain why the last value is what it is.

## I can…
Tick your four "I can…". Tell me what your variable stored.

> 🧑‍🏫 Note who traced the variable correctly. Homework: 4 multiple-choice questions.
