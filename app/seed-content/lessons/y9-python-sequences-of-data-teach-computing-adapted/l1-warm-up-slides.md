# Warm up — selection and lists

## Today we are learning
- read a program and predict what it shows
- use selection (`if` / `elif` / `else`)
- find and fix common syntax errors
- create a list and read an item by its index

> 🧑‍🏫 Read the four "I can…" aloud. Vocabulary on the board: selection · condition · syntax error · list · index. Same routine as every lesson.

## We work in pairs
![Two pupils pair programming at one computer]({{res:l1-pair-programming-teach-computing.png}})

> 🧑‍🏫 Pair programming: one driver (types), one navigator (reads and checks). Swap roles when I say. This is how we work all unit.

## Starter — let me check
Match each Python word to what it does.

> 🧑‍🏫 Use the matching task on the starter worksheet. Stress `=` (assign) vs `==` (check if equal) — this is the most common bug.

## Selection recap — if / elif / else
![A Python program that checks the day of the week with if-else]({{res:l1-weekday-selection-code-teach-computing.png}})

> 🧑‍🏫 Walk through it line by line. Days are numbers 0–6. The colon and the indent decide what is "inside" each branch.

## Spot the syntax errors  (we do)
A missing colon, `=` instead of `==`, or a missing quote will stop the program.

> 🧑‍🏫 Use the card-sort on the activity worksheet. Likely error: `if day = 4:` — fix-words "use `==` to compare, `=` to store."
