# Structuring a branching database

## Today we are learning
- write a yes/no question for a given attribute
- compare two branching databases
- explain why the order of questions matters

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: structure · attribute · question · order · even · efficient.

## Questions come from attributes
- every yes/no question is about an attribute
- attribute "wings" → "Does it have wings?"
- attribute "legs" → "Does it have legs?"

> 🧑‍🏫 The starter worksheet matches attributes to questions. Take a few pupil questions for the attribute "number of legs".

## Two trees, same objects
![A minibeast branching database]({{res:l4-minibeast-branching-database-teach-computing.png}})

- the EVEN tree splits each group roughly in half — short and fast
- a ONE-AT-A-TIME tree picks off one object each question — long and slow

> 🧑‍🏫 Both trees work, but the even one is better structured: you never ask more than a few questions. The activity worksheet compares them.

## Order matters
- ask the question that splits the WHOLE group in half first
- then split each smaller group

> 🧑‍🏫 Likely error: starting with "Is it the spider?" — that only picks out one object. Fix-words: start with a question that splits the whole group in two.

## Your turn
- order the questions so groups stay even
- explain which question you would ask first, and why

> 🧑‍🏫 Support: choose which tree is shorter. Core: count the questions on the longest path. Challenge: explain why even groups need fewer questions. TA: prompt, do not do it for them.
