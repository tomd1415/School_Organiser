# Creating a branching database

## Today we are learning
- use a branching database to identify an object
- build a branching database online with my own questions
- test my branching database to check it works

> 🧑‍🏫 Read the three "I can…" aloud. Vocabulary on the board: branching database · attribute · question · identify · test. Have j2data Branch open: ncce.io/branchingdb.

## Identify a minibeast
![A minibeast branching database]({{res:l3-minibeast-branching-database-teach-computing.png}})

Answer each yes/no question to reach the right minibeast.

> 🧑‍🏫 Use the example tree (ncce.io/minibtree). Together identify a caterpillar and a ladybird, reading each question and answering yes/no. The starter worksheet finds the bee.

## Build your own — j2data Branch
- choose a library set, click 6–8 objects
- press Sort, type a yes/no question
- drag objects into thumbs-up / thumbs-down
- add questions until each object is on its own

> 🧑‍🏫 Demonstrate one split live (e.g. food: "Is it fruit?"). Likely error: a question that does not split the objects — try to split the group into two even halves. Support: build with a partner using a given first question.

## Test it
- press Play
- pick an object and answer the questions to reach it
- swap with a partner and test theirs

> 🧑‍🏫 Pupils screenshot their finished tree onto the activity worksheet. TA: prompt, do not do it for them.
